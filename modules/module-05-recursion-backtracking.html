📖 Theory

🔄 The Recursion Template

Recursion is a method where the solution to a problem depends on solutions to smaller instances of the same problem. Backtracking is an algorithmic technique for solving problems recursively by trying to build a solution incrementally, one piece at a time, removing those solutions that fail to satisfy the constraints.

⚠️ The 3 Pillars of Recursion
  1. Base Case: The condition under which the recursion ends. Without this, you get a Stack Overflow!
  2. Action: What the current function call does (e.g., placing a queen, taking a number).
  3. Recursive Call: Calling the function on a smaller subset of data.

For Backtracking, add a 4th pillar: Undo Action (removing the queen, popping the number) immediately after the recursive call returns.

📝 Teaching Notes

Suggested Teaching Sequence
  1. Begin with the theory section — walk through key concepts (~20 min)
  2. Use the interactive visualization to demonstrate the algorithm (~15 min)
  3. Solve the first problem together as a class (~20 min)
  4. Assign remaining Easy problems as lab work, Medium/Hard as homework
Common Student Mistakes
  • Not handling edge cases (empty input, single element, duplicates)
  • Confusing time and space complexity of different approaches
  • Jumping to code before understanding the algorithm logic
Discussion Questions
  • What is the brute force approach and why is it insufficient?
  • How does the optimal approach improve on the brute force?
  • Can you identify the pattern that connects this module's problems?
