Climbing Stairs

Easy 🔗 LeetCode

📜 Problem Statement

You are climbing a staircase. It takes n steps to reach the top. Each time you can either climb 1 or 2 steps. In how many distinct ways can you climb to the top?

📚 Mathematical Foundation

To reach the i-th step, you could only have come from either the (i-1)-th step (by taking a 1-step) or the (i-2)-th step (by taking a 2-step).

Therefore, the total number of distinct ways to reach step i is the sum of the ways to reach step i-1 and step i-2. This is precisely the Fibonacci sequence.

Ways(n) = Ways(n-1) + Ways(n-2)

Base cases: Ways(1) = 1, Ways(2) = 2.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Brute Force Recursion

Directly translate the recurrence into a recursive function. Recomputes overlapping subproblems exponentially.

function climbStairs(n):
  if n <= 2: return n
  return climbStairs(n-1) + climbStairs(n-2)
Time: O(2n) Space: O(n)
Pros
  • Simple math translation
Cons
  • Time Limit Exceeded
Why optimize?
Redundant work. We calculate climbStairs(3) multiple times. We should cache the results!
Approach 2
Top-Down Memoization

Store the result of each step in a hash map or array. If we've seen it before, return it instantly.

function climbStairs(n, memo):
  if n <= 2: return n
  if memo[n] exists: return memo[n]
  memo[n] = climbStairs(n-1, memo) + climbStairs(n-2, memo)
  return memo[n]
Time: O(n) Space: O(n)
Pros
  • Linear time
Cons
  • Call stack overhead
Why optimize?
Recursion uses the call stack. We can iterate bottom-up instead using a DP table.
Approach 3
Bottom-Up Tabulation (1D DP)

Create an array dp of size n+1. Fill it iteratively starting from 1 up to n.

function climbStairs(n):
  if n <= 2: return n
  dp = array of size n+1
  dp[1] = 1, dp[2] = 2
  for i = 3 to n:
    dp[i] = dp[i-1] + dp[i-2]
  return dp[n]
Time: O(n) Space: O(n)
Pros
  • No stack overflow risk
Cons
  • O(n) space
Why optimize?
We only ever look at dp[i-1] and dp[i-2]. We don't need the entire array, just two variables!
Approach 4
Space-Optimized DP (Two Variables)

Keep track of just the last two calculated steps (like the Fibonacci sequence calculation).

function climbStairs(n):
  if n <= 2: return n
  a = 1, b = 2
  for i = 3 to n:
    c = a + b
    a = b
    b = c
  return b
Time: O(n) Space: O(1)
Pros
  • O(1) optimal space
  • Fastest execution
Cons
  • None

💻 Optimal Implementation

class Solution {
public:
    int climbStairs(int n) {
        if (n <= 2) return n;
        
        int prev2 = 1; // Step 1
        int prev1 = 2; // Step 2
        
        for (int i = 3; i <= n; i++) {
            int current = prev1 + prev2;
            prev2 = prev1;
            prev1 = current;
        }
        
        return prev1;
    }
};
class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 2:
            return n
            
        prev2 = 1
        prev1 = 2
        
        for _ in range(3, n + 1):
            current = prev1 + prev2
            prev2 = prev1
            prev1 = current
            
        return prev1
