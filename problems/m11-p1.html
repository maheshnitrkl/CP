Binary Tree Preorder Traversal

Easy 🔗 LeetCode

📜 Problem Statement

Given the root of a binary tree, return the preorder traversal of its nodes' values.

📚 Logic & Core Concept

Preorder Traversal visits the nodes of a binary tree in the following order:

  1. Visit the Current Node (Root).
  2. Traverse the Left Subtree.
  3. Traverse the Right Subtree.
Order: Node → Left → Right (NLR)

💡 Approach Progression: Brute Force → Optimal

Approach 1
Recursive Traversal

The most natural way to implement tree traversals. The function calls itself on the left and right children, naturally using the call stack.

function preorder(node, result):
  if node is null: return
  result.append(node.val) // Node
  preorder(node.left, result) // Left
  preorder(node.right, result) // Right
Time: O(n) Space: O(h) (Call Stack)
Pros
  • Extremely clean and readable
Cons
  • Risk of stack overflow for very deep skewed trees
Why optimize?
To avoid relying on the system call stack, we can manually simulate the recursion using our own Stack data structure.
Approach 2
Iterative (Using a Stack)

We use a LIFO (Last-In-First-Out) stack. To visit the left child before the right child, we must push the right child first, then the left child onto the stack.

function preorderIterative(root):
  if root is null: return []
  stack = [root]
  result = []
  while stack is not empty:
    node = stack.pop()
    result.append(node.val)
    if node.right exists: stack.push(node.right)
    if node.left exists: stack.push(node.left)
  return result
Time: O(n) Space: O(h)
Pros
  • No call stack overflow issues
Cons
  • Still requires O(h) extra memory
Why optimize?
Can we traverse the tree without any extra space? Yes, by temporarily modifying the tree structure using Morris Traversal!
Approach 3
Morris Traversal

Morris Traversal achieves O(1) space by creating temporary threaded links from the rightmost node of the left subtree back to the current node. This allows us to return to the parent without a stack.

function morrisPreorder(root):
  curr = root, result = []
  while curr != null:
    if curr.left == null:
      result.append(curr.val)
      curr = curr.right
    else:
      prev = curr.left
      while prev.right != null and prev.right != curr:
        prev = prev.right
      if prev.right == null: // Create thread
        result.append(curr.val)
        prev.right = curr
        curr = curr.left
      else: // Remove thread
        prev.right = null
        curr = curr.right
Time: O(n) Space: O(1)
Pros
  • Optimal O(1) extra space!
Cons
  • Temporarily mutates the tree
  • Complex to write and read

💻 Optimal Implementation

class Solution {
public:
    vector<int> preorderTraversal(TreeNode* root) {
        vector<int> result;
        if (root == nullptr) return result;
        
        stack<TreeNode*> s;
        s.push(root);
        
        while (!s.empty()) {
            TreeNode* node = s.top();
            s.pop();
            
            result.push_back(node->val);
            
            // Push right child first so left is processed first (LIFO)
            if (node->right) {
                s.push(node->right);
            }
            if (node->left) {
                s.push(node->left);
            }
        }
        
        return result;
    }
};
class Solution:
    def preorderTraversal(self, root: Optional[TreeNode]) -> List[int]:
        if not root:
            return []
            
        result = []
        stack = [root]
        
        while stack:
            node = stack.pop()
            result.append(node.val)
            
            # Push right child first so left is processed first (LIFO)
            if node.right:
                stack.append(node.right)
            if node.left:
                stack.append(node.left)
                
        return result
