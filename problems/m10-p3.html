House Robber

Medium 🔗 LeetCode

📜 Problem Statement

You are a professional robber planning to rob houses along a street. Each house has a certain amount of money stashed, the only constraint stopping you from robbing each of them is that adjacent houses have security systems connected and it will automatically contact the police if two adjacent houses were broken into on the same night. Given an integer array nums representing the amount of money of each house, return the maximum amount of money you can rob tonight without alerting the police.

📚 Mathematical Foundation

Let dp[i] be the maximum amount of money you can rob from the first i houses.

At house i, you have two choices:

  • Rob it: You get nums[i], but you couldn't have robbed house i-1. So you add dp[i-2].
  • Skip it: You keep whatever maximum money you had at house i-1, which is dp[i-1].
dp[i] = max(dp[i-1], nums[i] + dp[i-2])

💡 Approach Progression: Brute Force → Optimal

Approach 1
Brute Force Recursion

At every house, explore both options (rob or skip). Returns the max of both branches.

function robFrom(i, nums):
  if i < 0: return 0
  return max(robFrom(i-1, nums), nums[i] + robFrom(i-2, nums))
Time: O(2n) Space: O(n)
Pros
  • Pure translation of the recurrence
Cons
  • Exponential time, TLE
Why optimize?
We are solving the `robFrom(i)` problem multiple times. We can memoize the results!
Approach 2
Top-Down Memoization

Cache the maximum money we can get starting/ending at house i.

function robFrom(i, nums, memo):
  if i < 0: return 0
  if memo[i] exists: return memo[i]
  memo[i] = max(robFrom(i-1), nums[i] + robFrom(i-2))
  return memo[i]
Time: O(n) Space: O(n)
Pros
  • Linear time
Cons
  • Recursion uses call stack
Why optimize?
Avoid stack depth limits by converting to an iterative Tabulation approach.
Approach 3
Bottom-Up Tabulation (1D DP)

Build a dp array of size N where dp[i] is the max money up to house i.

function rob(nums):
  if nums.length == 0: return 0
  if nums.length == 1: return nums[0]
  dp = array of size N
  dp[0] = nums[0]
  dp[1] = max(nums[0], nums[1])
  for i = 2 to N-1:
    dp[i] = max(dp[i-1], nums[i] + dp[i-2])
  return dp[N-1]
Time: O(n) Space: O(n)
Pros
  • No stack limit risk
Cons
  • Allocates an entire array
Why optimize?
Since dp[i] only relies on dp[i-1] and dp[i-2], we only need to store two variables instead of a whole array!
Approach 4
Space-Optimized DP (Two Variables)

Track only the max money for the previous two houses (prev1 and prev2).

function rob(nums):
  prev1 = 0, prev2 = 0
  for x in nums:
    current = max(prev1, x + prev2)
    prev2 = prev1
    prev1 = current
  return prev1
Time: O(n) Space: O(1)
Pros
  • Optimal time and space
  • Extremely concise
Cons
  • None

💻 Optimal Implementation

class Solution {
public:
    int rob(vector<int>& nums) {
        int prev2 = 0; // dp[i-2]
        int prev1 = 0; // dp[i-1]
        
        for (int x : nums) {
            int current = max(prev1, x + prev2);
            prev2 = prev1;
            prev1 = current;
        }
        
        return prev1;
    }
};
class Solution:
    def rob(self, nums: List[int]) -> int:
        prev2, prev1 = 0, 0
        
        for x in nums:
            # current max is max of (skipping, robbing)
            current = max(prev1, x + prev2)
            prev2 = prev1
            prev1 = current
            
        return prev1
