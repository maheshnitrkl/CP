Binary Tree Postorder Traversal

Easy 🔗 LeetCode

📜 Problem Statement

Given the root of a binary tree, return the postorder traversal of its nodes' values.

📚 Logic & Core Concept

Postorder Traversal visits the nodes of a binary tree in the following order:

  1. Traverse the Left Subtree.
  2. Traverse the Right Subtree.
  3. Visit the Current Node (Root).
Order: Left → Right → Node (LRN)

Key Insight: Postorder is commonly used when you need to process children before their parent (e.g., deleting a tree, calculating directory sizes).

💡 Approach Progression: Brute Force → Optimal

Approach 1
Recursive Traversal

Like other traversals, recursion is the most elegant solution. The function recursively calls itself for the left child, then the right child, and finally processes the root node.

function postorder(node, result):
  if node is null: return
  postorder(node.left, result) // Left
  postorder(node.right, result) // Right
  result.append(node.val) // Node
Time: O(n) Space: O(h) (Call Stack)
Pros
  • Simple and elegant code
Cons
  • Subject to recursion depth limitations
Why optimize?
To solve the problem iteratively, postorder is famously harder than preorder and inorder because we must visit the parent node twice but only output it the second time. Let's see a clever 2-stack approach first.
Approach 2
Iterative (Using 2 Stacks)

We know Preorder is NLR. If we tweak Preorder to do NRL, and then reverse the result, we get exactly LRN! We can use a second stack (or an array reversed at the end) to store the NRL sequence.

function postorder2Stacks(root):
  if root is null: return []
  stack1 = [root], stack2 = []
  while stack1 is not empty:
    node = stack1.pop()
    stack2.push(node)
    if node.left exists: stack1.push(node.left)
    if node.right exists: stack1.push(node.right)
  return stack2 reversed
Time: O(n) Space: O(n) (For both stacks)
Pros
  • Very easy to understand and write
Cons
  • Uses O(n) extra space instead of O(h)
Why optimize?
Can we do it iteratively with only ONE stack, bringing the space complexity down to O(h)? Yes, by tracking the previously visited node!
Approach 3
Iterative (Using 1 Stack)

Traverse as far left as possible. When you reach a null left child, peek the stack. If the right child exists and hasn't been visited, traverse right. Otherwise, pop and process the node.

function postorder1Stack(root):
  stack = [], result = []
  curr = root, lastVisited = null
  while curr != null or stack is not empty:
    if curr != null:
      stack.push(curr)
      curr = curr.left
    else:
      peekNode = stack.peek()
      if peekNode.right != null and lastVisited != peekNode.right:
        curr = peekNode.right
      else:
        result.append(peekNode.val)
        lastVisited = stack.pop()
Time: O(n) Space: O(h)
Pros
  • Optimal iterative space O(h)
Cons
  • Tricky to get the logic perfectly right

💻 Optimal Implementation

class Solution {
public:
    vector<int> postorderTraversal(TreeNode* root) {
        vector<int> result;
        if (!root) return result;
        
        stack<TreeNode*> s;
        TreeNode* curr = root;
        TreeNode* lastVisited = nullptr;
        
        while (curr != nullptr || !s.empty()) {
            if (curr != nullptr) {
                s.push(curr);
                curr = curr->left;
            } else {
                TreeNode* peekNode = s.top();
                // If right child exists and traversing node
                // from left child, then move right
                if (peekNode->right != nullptr && lastVisited != peekNode->right) {
                    curr = peekNode->right;
                } else {
                    result.push_back(peekNode->val);
                    lastVisited = s.top();
                    s.pop();
                }
            }
        }
        
        return result;
    }
};
class Solution:
    def postorderTraversal(self, root: Optional[TreeNode]) -> List[int]:
        result = []
        if not root:
            return result
            
        stack = []
        curr = root
        last_visited = None
        
        while curr or stack:
            if curr:
                stack.append(curr)
                curr = curr.left
            else:
                peek_node = stack[-1]
                # If right child exists and traversing node
                # from left child, then move right
                if peek_node.right and last_visited != peek_node.right:
                    curr = peek_node.right
                else:
                    result.append(peek_node.val)
                    last_visited = stack.pop()
                    
        return result
