Diameter of Binary Tree

Easy 🔗 LeetCode

📜 Problem Statement

Given the root of a binary tree, return the length of the diameter of the tree. The diameter of a binary tree is the length of the longest path between any two nodes in a tree. This path may or may not pass through the root.

📚 Logic & Core Concept

The diameter of a binary tree is the length of the longest path between any two nodes. The path may or may not pass through the root.

Length is defined by the number of edges. So a path with 3 nodes has a length of 2.

Diameter at Node = Depth(node.left) + Depth(node.right)

The overall diameter of the tree is the maximum diameter found at any node in the tree.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Brute Force (Call Depth for Every Node)

For every node, we calculate the depth of its left and right subtrees, sum them up as a potential diameter, and then recursively call diameter() on left and right children to find the absolute maximum.

function diameterOfBinaryTree(root):
  if root is null: return 0
  currentDiameter = depth(root.left) + depth(root.right)
  leftDiameter = diameterOfBinaryTree(root.left)
  rightDiameter = diameterOfBinaryTree(root.right)
  return max(currentDiameter, max(leftDiameter, rightDiameter))
Time: O(n2) Space: O(h)
Pros
  • Straightforward logic based directly on the definition
Cons
  • Very slow. We recalculate depth repeatedly for the same nodes.
Why optimize?
We can calculate the diameter while we are calculating the depth in a single bottom-up traversal!
Approach 2
Optimized Bottom-Up DFS O(n)

We maintain a global/reference variable max_diameter. We modify our standard depth() function: before a node returns its depth to its parent, it calculates left_depth + right_depth and updates max_diameter if it's larger.

function diameterOfBinaryTree(root):
  max_d = 0
  function depth(node):
    if node is null: return 0
    left = depth(node.left)
    right = depth(node.right)
    max_d = max(max_d, left + right) // Update global max
    return 1 + max(left, right) // Return depth
  depth(root)
  return max_d
Time: O(n) Space: O(h)
Pros
  • Visits each node exactly once
  • Elegant modification of a simple algorithm
Cons
  • Requires a global/reference variable to carry state

💻 Optimal Implementation

class Solution {
    int max_diameter = 0;
    
    int depth(TreeNode* root) {
        if (root == nullptr) return 0;
        
        int left_depth = depth(root->left);
        int right_depth = depth(root->right);
        
        // Diameter passing through this node is sum of depths
        max_diameter = max(max_diameter, left_depth + right_depth);
        
        // Return max depth from this node
        return 1 + max(left_depth, right_depth);
    }
    
public:
    int diameterOfBinaryTree(TreeNode* root) {
        max_diameter = 0; // reset for multiple test cases
        depth(root);
        return max_diameter;
    }
};
class Solution:
    def diameterOfBinaryTree(self, root: Optional[TreeNode]) -> int:
        self.max_diameter = 0
        
        def depth(node):
            if not node:
                return 0
                
            left_depth = depth(node.left)
            right_depth = depth(node.right)
            
            # Diameter passing through this node is sum of depths
            self.max_diameter = max(self.max_diameter, left_depth + right_depth)
            
            # Return max depth from this node
            return 1 + max(left_depth, right_depth)
            
        depth(root)
        return self.max_diameter
