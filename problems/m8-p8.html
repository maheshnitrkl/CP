Sliding Window Maximum

Hard 🔗 LeetCode

📜 Problem Statement

Given array nums and window size k, return the max sliding window array.

🔢 Mathematical Foundation

Monotonic Deque

Maintain a deque of indices. Keep it strictly decreasing in values. If a new element is larger than elements at the back, they can never be the max — pop them. Remove elements from the front if they fall out of the window.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Brute Force Window Scan

For every window of size k, iterate through all k elements to find the maximum.

function maxSlidingWindow(nums, k):
  n = len(nums), res = []
  for i = 0 to n-k:
    max_val = -infinity
    for j = i to i+k-1:
      max_val = max(max_val, nums[j])
    res.append(max_val)
  return res
Time: O(n × k) Space: O(1)
Pros
  • Trivial to write
Cons
  • Very slow when k is large
  • O(n*k) time limit exceeded
Why optimize?
We are recalculating the max for (k-1) overlapping elements. We should use a data structure to maintain the maximum efficiently.
Approach 2
Max-Heap / Priority Queue

Use a Max-Heap storing (value, index). The max is always at the top. If the top element's index is out of the current window, pop it.

function maxSlidingWindow(nums, k):
  heap = MaxHeap(), res = []
  for i = 0 to n-1:
    heap.push((nums[i], i))
    if i >= k - 1:
      while heap.peek().index <= i - k:
        heap.pop()
      res.append(heap.peek().value)
  return res
Time: O(n log k) Space: O(k)
Pros
  • Much faster than brute force
Cons
  • Heap operations cost O(log k)
Why optimize?
By keeping the elements strictly decreasing, we don't need a heap. We can maintain the maximum in O(1) amortized time using a Deque.
Approach 3
Monotonic Deque

Deque stores indices. Front of deque is the max for the current window.

function maxSlidingWindow(nums, k):
  dq = Deque(), res = []
  for i, x in enumerate(nums):
    // remove out of bounds
    if dq and dq.front() == i - k: dq.popFront()
    // remove smaller elements (useless)
    while dq and nums[dq.back()] < x: dq.popBack()
    dq.pushBack(i)
    if i >= k - 1: res.append(nums[dq.front()])
  return res
Time: O(n) Space: O(k)
Pros
  • O(n) time
  • Classic Deque usage
Cons
  • Implementation details can be tricky

💻 Optimal Implementation

class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        deque<int> dq; vector<int> res;
        for (int i = 0; i < nums.size(); i++) {
            if (!dq.empty() && dq.front() == i - k) dq.pop_front();
            while (!dq.empty() && nums[dq.back()] < nums[i]) dq.pop_back();
            dq.push_back(i);
            if (i >= k - 1) res.push_back(nums[dq.front()]);
        }
        return res;
    }
};
from collections import deque
class Solution:
    def maxSlidingWindow(self, nums, k):
        dq = deque(); res = []
        for i, x in enumerate(nums):
            if dq and dq[0] == i - k: dq.popleft()
            while dq and nums[dq[-1]] < x: dq.pop()
            dq.append(i)
            if i >= k - 1: res.append(nums[dq[0]])
        return res
