Majority Element

Easy 🔗 LeetCode

📜 Problem Statement

Given an array, find the element that appears more than ⌊n/2⌋ times. Guaranteed to exist.

Example: [3,2,3] → 3

🔢 Mathematical Foundation

Boyer-Moore Voting Algorithm

Maintain a candidate and a count. For each element: if count is 0, set new candidate. If same as candidate, increment count. Otherwise, decrement count.

The majority element survives because it appears more than all others combined. Think of it as "voting": the majority always wins when votes are paired against it.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Hash Map Counting

Count frequency of each element. Return the one exceeding n/2.

function majorityElement(nums):
  freq = HashMap()
  for x in nums: freq[x] += 1
  for key, count in freq:
    if count > n/2: return key
Time: O(n) Space: O(n)
Pros
  • Simple and correct
  • Works for any frequency threshold
Cons
  • O(n) extra space for the hash map
Why do we need a better approach?
Can we find the majority element without using any extra space? Boyer-Moore's voting algorithm does exactly that.
Approach 2
Sorting

Sort the array. The majority element must be at index n/2 (since it occupies more than half the array).

function majorityElement(nums):
  sort(nums)
  return nums[n/2]
Time: O(n log n) Space: O(1)
Pros
  • Very simple
  • No extra space
Cons
  • O(n log n) sorting is slower than linear
Why do we need a better approach?
Sorting is O(n log n). Boyer-Moore achieves O(n) time with O(1) space.
Approach 3
Boyer-Moore Voting

Pair each majority element vote against a non-majority vote. The majority survives because it has more votes than all others combined.

function majorityElement(nums):
  candidate = nums[0], count = 1
  for i = 1 to n-1:
    if count == 0:
      candidate = nums[i]; count = 1
    elif nums[i] == candidate:
      count += 1
    else:
      count -= 1
  return candidate
Time: O(n) Space: O(1)
Pros
  • O(n) time, O(1) space
  • Elegant voting analogy
  • No data structures needed
Cons
  • Only works when majority is guaranteed to exist
  • Requires verification pass if majority isn't guaranteed

💻 Optimal Implementation

class Solution {
public:
    int majorityElement(vector<int>& nums) {
        int candidate = nums[0], count = 1;
        for (int i = 1; i < nums.size(); i++) {
            if (count == 0) { candidate = nums[i]; count = 1; }
            else if (nums[i] == candidate) count++;
            else count--;
        }
        return candidate;
    }
};
class Solution:
    def majorityElement(self, nums):
        candidate, count = nums[0], 1
        for num in nums[1:]:
            if count == 0:
                candidate, count = num, 1
            elif num == candidate:
                count += 1
            else:
                count -= 1
        return candidate
