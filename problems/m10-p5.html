Unique Paths

Medium 🔗 LeetCode

📜 Problem Statement

There is a robot on an m x n grid. The robot is initially located at the top-left corner (i.e., grid[0][0]). The robot tries to move to the bottom-right corner (i.e., grid[m - 1][n - 1]). The robot can only move either down or right at any point in time. Given the two integers m and n, return the number of possible unique paths that the robot can take to reach the bottom-right corner.

📚 Mathematical Foundation

Let dp[r][c] be the number of unique paths from the top-left corner (0,0) to the cell (r,c).

Since the robot can only move down or right, it can only reach (r,c) from either:

  • The cell directly above it: (r-1, c)
  • The cell directly to its left: (r, c-1)
dp[r][c] = dp[r-1][c] + dp[r][c-1]

Base cases: The first row and first column only have 1 unique path (going straight right or straight down).

💡 Approach Progression: Brute Force → Optimal

Approach 1
Brute Force Recursion

Traverse the grid recursively. From every cell, explore moving right and moving down. Return 1 when the bottom-right corner is reached.

function paths(r, c, m, n):
  if r == m-1 and c == n-1: return 1
  if r ≥ m or c ≥ n: return 0
  return paths(r+1, c) + paths(r, c+1)
Time: O(2m+n) Space: O(m+n)
Pros
  • Simple DFS logic
Cons
  • Exponential time, TLE
Why optimize?
Many paths converge on the same grid cell. We calculate the number of paths from `(r, c)` to the end over and over.
Approach 2
2D Tabulation (DP Table)

Build an m × n matrix. Initialize the first row and column to 1, then calculate the inner cells.

function uniquePaths(m, n):
  dp = 2D array [m][n]
  for i = 0 to m-1: dp[i][0] = 1
  for j = 0 to n-1: dp[0][j] = 1
  for i = 1 to m-1:
    for j = 1 to n-1:
      dp[i][j] = dp[i-1][j] + dp[i][j-1]
  return dp[m-1][n-1]
Time: O(m × n) Space: O(m × n)
Pros
  • Fast polynomial time
Cons
  • O(m × n) memory is inefficient
Why optimize?
Notice that to calculate the current row, we only need values from the previous row. We can drop the 2D array!
Approach 3
1D DP (Space Optimized)

Maintain a 1D array representing the previous row. Update it in place as we scan through the rows.

function uniquePaths(m, n):
  dp = array [n] of 1s
  for i = 1 to m-1:
    for j = 1 to n-1:
      dp[j] = dp[j] + dp[j-1]
  return dp[n-1]
Time: O(m × n) Space: O(n)
Pros
  • Great space reduction
Cons
  • Still takes O(m × n) iterations
Why optimize?
This is actually a pure Combinatorics problem. We need to make a total of `m-1` down steps and `n-1` right steps, in any order!
Approach 4
Combinatorics (Math)

The total number of steps is (m-1) + (n-1) = m+n-2. We must choose m-1 down steps (or n-1 right steps). The answer is simply (m+n-2) Choose (m-1).

function uniquePaths(m, n):
  N = m + n - 2
  R = min(m - 1, n - 1)
  res = 1
  for i = 1 to R:
    res = res * (N - R + i) / i
  return res
Time: O(min(m, n)) Space: O(1)
Pros
  • O(1) space, incredibly fast
Cons
  • Requires integer overflow care

💻 Optimal Implementation

class Solution {
public:
    int uniquePaths(int m, int n) {
        // Total steps = m-1 + n-1 = m+n-2
        // We choose m-1 steps to go down (or n-1 to go right)
        int N = n + m - 2;
        int R = min(m - 1, n - 1);
        double res = 1;
        
        // Calculate (N choose R) safely
        for (int i = 1; i <= R; i++) {
            res = res * (N - R + i) / i;
        }
        
        return (int)res;
    }
};
class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # Space optimized DP using a single 1D array
        dp = [1] * n
        
        # Traverse rows
        for i in range(1, m):
            # Traverse columns
            for j in range(1, n):
                # dp[j] gets updated with itself (cell above) + left cell
                dp[j] = dp[j] + dp[j - 1]
                
        return dp[-1]
