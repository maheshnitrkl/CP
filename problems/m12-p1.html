DFS Traversal

Easy 🔗 Problem Link

📜 Problem Statement

Given a connected undirected graph, perform a Depth First Search (DFS) traversal starting from vertex 0.

📚 Logic & Core Concept

Depth First Search (DFS) is a fundamental graph traversal algorithm. The concept is simple: go deep before going wide.

Starting from a given vertex, you explore as far as possible along each branch before backtracking. To prevent infinite loops in graphs with cycles, we must keep track of which vertices have already been visited.

Key Components:
  • Visited Array: A boolean array to track nodes we've seen.
  • Stack/Call Stack: To remember where to backtrack when a path ends.

💡 Approach Progression

Approach 1
Recursive DFS (Standard)

We use the system's call stack to traverse the graph recursively. When we visit a node, we mark it as visited, then recursively call DFS on all its unvisited neighbors.

function dfs(node, visited, adj, result):
  visited[node] = true
  result.append(node)
  
  for neighbor in adj[node]:
    if not visited[neighbor]:
      dfs(neighbor, visited, adj, result)
Time: O(V + E) Space: O(V)
Pros
  • Very concise and easy to write.
  • Standard way to write DFS in competitive programming.
Cons
  • Can cause stack overflow for extremely deep graphs (rarely an issue within standard constraints).
Why explore alternatives?
If recursion depth is a hard constraint (e.g. strict system limits), an explicit stack avoids call stack overflows.
Approach 2
Iterative DFS (Explicit Stack)

We simulate the recursion stack using our own data structure. Note that to match the exact traversal order of recursive DFS (which processes neighbors from left to right), we would need to push neighbors to the stack in reverse order.

function dfs_iterative(start_node, adj):
  stack = [start_node]
  visited = array of false
  result = []
  
  while stack is not empty:
    curr = stack.pop()
    
    if not visited[curr]:
      visited[curr] = true
      result.append(curr)
      // Push neighbors in reverse order to match recursive DFS
      for i from len(adj[curr])-1 down to 0:
        neighbor = adj[curr][i]
        if not visited[neighbor]:
          stack.push(neighbor)
  return result
Time: O(V + E) Space: O(V)
Pros
  • No risk of call stack overflow.
Cons
  • Slightly more code.
  • Managing traversal order (reversing neighbors) is prone to bugs.

💻 Optimal Implementation

class Solution {
private:
    void dfs(int node, vector<int> adj[], vector<bool>& visited, vector<int>& result) {
        // Mark the current node as visited and add it to result
        visited[node] = true;
        result.push_back(node);
        
        // Traverse all its neighbors
        for (int neighbor : adj[node]) {
            if (!visited[neighbor]) {
                dfs(neighbor, adj, visited, result);
            }
        }
    }
    
public:
    // Function to return a list containing the DFS traversal of the graph.
    vector<int> dfsOfGraph(int V, vector<int> adj[]) {
        vector<int> result;
        vector<bool> visited(V, false);
        
        // Start DFS from node 0
        dfs(0, adj, visited, result);
        
        return result;
    }
};
class Solution:
    # Function to return a list containing the DFS traversal of the graph.
    def dfsOfGraph(self, V: int, adj: List[List[int]]) -> List[int]:
        result = []
        visited = [False] * V
        
        def dfs(node):
            # Mark the current node as visited and add it to result
            visited[node] = True
            result.append(node)
            
            # Traverse all its neighbors
            for neighbor in adj[node]:
                if not visited[neighbor]:
                    dfs(neighbor)
                    
        # Start DFS from node 0
        dfs(0)
        
        return result
