Course Schedule

Medium 🔗 Problem Link

📜 Problem Statement

There are a total of numCourses courses you have to take. You are given an array prerequisites where prerequisites[i] = [ai, bi] indicates that you must take course bi first if you want to take course ai. Return true if you can finish all courses. Otherwise, return false.

📚 Logic & Core Concept

You have to take numCourses courses, labeled 0 to numCourses-1. You are given an array of prerequisites where prerequisites[i] = [a, b] means you must take course b before you can take course a.

This is a classic problem of finding if a Directed Graph has a cycle. If there's a cycle (e.g., A requires B, and B requires A), you can never finish all courses.

There are two standard ways to detect cycles in a directed graph:
  • DFS Cycle Detection (3-Colors): Mark nodes as unvisited, visiting (in current path), and visited (fully processed). If we hit a 'visiting' node, we found a cycle!
  • Kahn's Algorithm (BFS Topological Sort): Count how many prerequisites each course has (in-degree). Start with courses that have 0 prerequisites. Remove them, decrease the in-degree of their neighbors, and repeat. If we can process all courses, there's no cycle.

💡 Approach Progression: DFS → Kahn's Algorithm

Approach 1
DFS Cycle Detection (3 States)

We maintain a state array for each node: 0 = unvisited, 1 = visiting (currently on the recursion stack), 2 = visited (fully processed and safe). If during our DFS we ever encounter a node with state 1, it means we looped back to a node currently in our path—a cycle!

function canFinishDFS(numCourses, prerequisites):
  adj = buildAdjacencyList(numCourses, prerequisites)
  state = array of 0s (size numCourses)
  
  function hasCycle(node):
    if state[node] == 1: return true // Cycle detected!
    if state[node] == 2: return false // Already checked and safe
    
    state[node] = 1 // Mark as visiting
    for neighbor in adj[node]:
      if hasCycle(neighbor): return true
    
    state[node] = 2 // Mark as fully processed
    return false
  
  for i from 0 to numCourses - 1:
    if state[i] == 0 and hasCycle(i):
      return false // Found a cycle
  return true // No cycles found
Time: O(V + E) Space: O(V + E) (Adj List + Recursion Stack)
Approach 2
Kahn's Algorithm (Topological Sort via BFS)

Instead of recursion, we count the "in-degree" (number of incoming edges/prerequisites) for each node. We put all nodes with 0 in-degree into a queue. We process them, reducing the in-degree of their neighbors by 1. If a neighbor reaches 0, it goes in the queue. If the total processed nodes equals numCourses, we succeed.

function canFinishKahn(numCourses, prerequisites):
  adj = buildAdjacencyList(numCourses, prerequisites)
  in_degree = array of 0s
  
  for (course, prereq) in prerequisites:
    in_degree[course] += 1
  
  queue = queue of all courses with in_degree == 0
  count = 0
  
  while queue is not empty:
    curr = queue.dequeue()
    count += 1
    for neighbor in adj[curr]:
      in_degree[neighbor] -= 1
      if in_degree[neighbor] == 0:
        queue.enqueue(neighbor)
  
  return count == numCourses
Time: O(V + E) Space: O(V + E)
Pros
  • Iterative (no recursion limits).
  • Naturally produces a valid topological ordering (useful if the problem asks for the actual schedule).
Cons
  • Requires maintaining an extra array (`in_degree`).

💻 Optimal Implementation

class Solution {
public:
    bool canFinish(int numCourses, vector<vector<int>>& prerequisites) {
        vector<vector<int>> adj(numCourses);
        vector<int> inDegree(numCourses, 0);
        
        // Build graph
        for (const auto& pre : prerequisites) {
            adj[pre[1]].push_back(pre[0]);
            inDegree[pre[0]]++;
        }
        
        queue<int> q;
        // Enqueue courses with 0 prerequisites
        for (int i = 0; i < numCourses; i++) {
            if (inDegree[i] == 0) {
                q.push(i);
            }
        }
        
        int count = 0;
        while (!q.empty()) {
            int curr = q.front();
            q.pop();
            count++;
            
            for (int neighbor : adj[curr]) {
                inDegree[neighbor]--;
                if (inDegree[neighbor] == 0) {
                    q.push(neighbor);
                }
            }
        }
        
        return count == numCourses;
    }
};
class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        adj = collections.defaultdict(list)
        in_degree = [0] * numCourses
        
        # Build graph
        for crs, pre in prerequisites:
            adj[pre].append(crs)
            in_degree[crs] += 1
            
        # Queue for nodes with 0 in-degree
        queue = collections.deque([i for i in range(numCourses) if in_degree[i] == 0])
        count = 0
        
        while queue:
            curr = queue.popleft()
            count += 1
            
            for neighbor in adj[curr]:
                in_degree[neighbor] -= 1
                if in_degree[neighbor] == 0:
                    queue.append(neighbor)
                    
        return count == numCourses
class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        adj = collections.defaultdict(list)
        for crs, pre in prerequisites:
            adj[pre].append(crs)
            
        # 0: unvisited, 1: visiting, 2: visited
        state = [0] * numCourses
        
        def hasCycle(crs):
            if state[crs] == 1:
                return True
            if state[crs] == 2:
                return False
                
            state[crs] = 1 # Mark as visiting
            for neighbor in adj[crs]:
                if hasCycle(neighbor):
                    return True
            
            state[crs] = 2 # Mark as fully processed
            return False
            
        for i in range(numCourses):
            if state[i] == 0:
                if hasCycle(i):
                    return False
                    
        return True
