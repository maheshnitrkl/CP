Minimum Path Sum

Medium 🔗 LeetCode

📜 Problem Statement

Given a m x n grid filled with non-negative numbers, find a path from top left to bottom right, which minimizes the sum of all numbers along its path. Note: You can only move either down or right at any point in time.

📚 Mathematical Foundation

Let dp[r][c] be the minimum path sum from the top-left corner (0,0) to the cell (r,c).

Since the robot can only move down or right, the minimum cost to reach (r,c) is the cost of the current cell plus the minimum of the paths to its top and left neighbors:

dp[r][c] = grid[r][c] + min(dp[r-1][c], dp[r][c-1])

Base cases:

  • dp[0][0] = grid[0][0]
  • First row: dp[0][c] = grid[0][c] + dp[0][c-1]
  • First column: dp[r][0] = grid[r][0] + dp[r-1][0]

💡 Approach Progression: Brute Force → Optimal

Approach 1
Brute Force Recursion

Traverse the grid recursively. From every cell, explore moving right and moving down. Return the minimum path sum among all choices.

function minPath(r, c, grid):
  if r ≥ m or c ≥ n: return INF
  if r == m-1 and c == n-1: return grid[r][c]
  return grid[r][c] + min(minPath(r+1, c), minPath(r, c+1))
Time: O(2m+n) Space: O(m+n)
Pros
  • Simple DFS logic
Cons
  • Exponential time, TLE
Why optimize?
Many paths converge on the same grid cell. We calculate the minimum path from `(r, c)` to the end over and over.
Approach 2
Top-Down Memoization

Use a 2D array or hash map to cache the minimum path from (r,c) to the destination.

function minPath(r, c, grid, memo):
  if r ≥ m or c ≥ n: return INF
  if r == m-1 and c == n-1: return grid[r][c]
  if memo[r][c] exists: return memo[r][c]
  memo[r][c] = grid[r][c] + min(minPath(r+1, c), minPath(r, c+1))
  return memo[r][c]
Time: O(m × n) Space: O(m × n)
Pros
  • Fast pseudo-polynomial time
Cons
  • Recursion uses call stack
Why optimize?
We can avoid recursion stack limits by doing this iteratively from the start to the end (Tabulation).
Approach 3
2D DP Tabulation

Create a 2D dp array of size m x n. Fill it row by row, column by column.

function minPathSum(grid):
  dp = 2D array [m][n]
  dp[0][0] = grid[0][0]
  for i = 1 to m-1: dp[i][0] = grid[i][0] + dp[i-1][0]
  for j = 1 to n-1: dp[0][j] = grid[0][j] + dp[0][j-1]
  for i = 1 to m-1:
    for j = 1 to n-1:
      dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])
  return dp[m-1][n-1]
Time: O(m × n) Space: O(m × n)
Pros
  • Iterative, avoids stack limit
Cons
  • Allocates a new O(m*n) matrix
Why optimize?
We don't actually need a separate `dp` array. If we are allowed to modify the input grid, we can do it in O(1) extra space!
Approach 4
In-Place DP (Modify Input)

Use the input grid itself as the DP table. We just overwrite grid[i][j] with the minimum sum to reach it.

function minPathSum(grid):
  for i = 0 to m-1:
    for j = 0 to n-1:
      if i == 0 and j == 0: continue
      if i == 0: grid[i][j] += grid[i][j-1]
      else if j == 0: grid[i][j] += grid[i-1][j]
      else: grid[i][j] += min(grid[i-1][j], grid[i][j-1])
  return grid[m-1][n-1]
Time: O(m × n) Space: O(1)
Pros
  • No extra memory used
Cons
  • Mutates input data (not always allowed)

💻 Optimal Implementation

class Solution {
public:
    int minPathSum(vector<vector<int>>& grid) {
        int m = grid.size();
        int n = grid[0].size();
        
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (i == 0 && j == 0) {
                    continue;
                } else if (i == 0) {
                    grid[i][j] += grid[i][j - 1]; // Only from left
                } else if (j == 0) {
                    grid[i][j] += grid[i - 1][j]; // Only from top
                } else {
                    grid[i][j] += min(grid[i - 1][j], grid[i][j - 1]); // Min of top and left
                }
            }
        }
        
        return grid[m - 1][n - 1];
    }
};
class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        m, n = len(grid), len(grid[0])
        
        for i in range(m):
            for j in range(n):
                if i == 0 and j == 0:
                    continue
                elif i == 0:
                    grid[i][j] += grid[i][j - 1] # Only from left
                elif j == 0:
                    grid[i][j] += grid[i - 1][j] # Only from top
                else:
                    # Minimum of the path from top or left
                    grid[i][j] += min(grid[i - 1][j], grid[i][j - 1])
                    
        return grid[-1][-1]
