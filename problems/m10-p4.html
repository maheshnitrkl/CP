Partition Equal Subset Sum

Medium 🔗 LeetCode

📜 Problem Statement

Given an integer array nums, return true if you can partition the array into two subsets such that the sum of the elements in both subsets is equal or false otherwise.

📚 Mathematical Foundation

This is a variation of the 0/1 Knapsack problem. If the total sum of the array is odd, we cannot divide it into two equal integer subsets. So, if sum % 2 != 0, return false immediately.

If the sum is even, our goal is to find a subset that adds up exactly to target = sum / 2.

Let dp[i][j] be a boolean indicating whether we can make sum j using a subset of the first i items.

  • Exclude item i: The subset sum remains j. We inherit dp[i-1][j].
  • Include item i: If j ≥ nums[i], the remaining sum we need is j - nums[i]. We check dp[i-1][j - nums[i]].
dp[i][j] = dp[i-1][j] OR dp[i-1][j - nums[i]]

💡 Approach Progression: Brute Force → Optimal

Approach 1
Backtracking (Recursion)

At each element, we either include it in the subset or exclude it. We recursively check if either path hits exactly target = sum / 2.

function canFindSum(i, currentSum, target, nums):
  if currentSum == target: return True
  if currentSum > target or i ≥ len(nums): return False
  include = canFindSum(i+1, currentSum + nums[i], target)
  exclude = canFindSum(i+1, currentSum, target)
  return include or exclude
Time: O(2n) Space: O(n)
Pros
  • Straightforward logic
Cons
  • Exponential time
Why optimize?
We evaluate the same (index, currentSum) state multiple times. We need to memoize these overlapping subproblems.
Approach 2
Top-Down Memoization

Use a 2D array or hash map memo[index][currentSum] to cache whether a sum is possible from a given index.

function canFind(i, sum_left, memo):
  if sum_left == 0: return True
  if i < 0 or sum_left < 0: return False
  if memo[i][sum_left] exists: return memo[i][sum_left]
  ans = canFind(i-1, sum_left) or canFind(i-1, sum_left - nums[i])
  memo[i][sum_left] = ans
  return ans
Time: O(n × target) Space: O(n × target)
Pros
  • Pseudo-polynomial time
Cons
  • Large memory footprint for recursion + 2D cache
Why optimize?
A bottom-up 2D DP Table avoids recursion overhead. And we can then optimize space from 2D to 1D.
Approach 3
2D Tabulation (0/1 Knapsack)

Build a 2D boolean array dp[n][target+1] iteratively.

function canPartition(nums):
  target = sum(nums) / 2
  dp = 2D array [N][target+1] of False
  for i = 0 to N-1: dp[i][0] = True
  for i = 1 to N-1:
    for j = 1 to target:
      if j ≥ nums[i]: dp[i][j] = dp[i-1][j] or dp[i-1][j-nums[i]]
      else: dp[i][j] = dp[i-1][j]
  return dp[N-1][target]
Time: O(n × target) Space: O(n × target)
Pros
  • No recursion stack
Cons
  • Still O(n × target) memory
Why optimize?
Notice that dp[i] only relies on the previous row dp[i-1]. We can flatten the 2D array into a 1D array.
Approach 4
1D Tabulation (Space Optimized)

Use a single 1D array dp[target+1]. To prevent overwriting data we need for the current iteration, we must iterate j backwards.

function canPartition(nums):
  target = sum(nums) / 2
  dp = array [target+1] of False
  dp[0] = True
  for x in nums:
    for j = target down to x:
      dp[j] = dp[j] or dp[j - x]
  return dp[target]
Time: O(n × target) Space: O(target)
Pros
  • Optimal space
  • Extremely fast inner loop
Cons
  • Iteration direction must be strictly backwards

💻 Optimal Implementation

class Solution {
public:
    bool canPartition(vector<int>& nums) {
        int sum = 0;
        for (int x : nums) {
            sum += x;
        }
        
        // If sum is odd, cannot partition into two equal subsets
        if (sum % 2 != 0) return false;
        
        int target = sum / 2;
        vector<bool> dp(target + 1, false);
        dp[0] = true; // We can always make sum 0 (by choosing empty subset)
        
        for (int x : nums) {
            // Traverse backwards to avoid using the same item multiple times
            for (int j = target; j >= x; j--) {
                if (dp[j - x]) {
                    dp[j] = true;
                }
            }
        }
        
        return dp[target];
    }
};
class Solution:
    def canPartition(self, nums: List[int]) -> bool:
        total = sum(nums)
        
        # If sum is odd, cannot partition into two equal subsets
        if total % 2 != 0:
            return False
            
        target = total // 2
        dp = [False] * (target + 1)
        dp[0] = True
        
        for x in nums:
            # Traverse backwards to avoid using the same item multiple times
            for j in range(target, x - 1, -1):
                if dp[j - x]:
                    dp[j] = True
                    
        return dp[target]
