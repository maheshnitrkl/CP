Subsets

Medium 🔗 LeetCode

📜 Problem Statement

Given an integer array nums of unique elements, return all possible subsets (the power set). The solution set must not contain duplicate subsets.

Example: nums = [1,2,3] → [[], [1], [2], [3], [1,2], [1,3], [2,3], [1,2,3]]

Key Insight: For n elements, there are exactly 2n subsets (each element is either included or excluded).

🔢 Mathematical Foundation

|Power Set| = 2n

Each element has 2 choices: include or exclude. With n elements, total subsets = 2 × 2 × ... × 2 = 2n.

This creates a binary decision tree of depth n. Each leaf represents one complete subset.

Bit Masking: Integer i from 0 to 2n-1 encodes a subset

Bit j of i being 1 means element j is included. Example: i=5 (101) → include elements at index 0 and 2.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Iterative (Cascading)

Start with an empty set. For each number, take all existing subsets and create new ones by adding the current number.

function subsets(nums):
  result = [[]] // start with empty set
  for num in nums:
    newSubsets = []
    for subset in result:
      newSubsets.append(subset + [num])
    result.extend(newSubsets)
  return result
Time: O(n × 2n) Space: O(n × 2n)
Pros
  • No recursion needed
  • Easy to understand iteratively
Cons
  • Creates many intermediate copies
  • Hard to extend to constrained variants (Subsets II)
Why do we need a better approach?
While this works, it's hard to adapt for problems with constraints (e.g., no duplicates, combination sum). Backtracking provides a universal template.
Approach 2
Backtracking (Include/Exclude Tree)

Build a binary decision tree. At each level (element), decide to include or exclude it. When we reach the end, we have a complete subset.

function subsets(nums):
  result = []
  function backtrack(start, current):
    result.append(copy(current)) // every node is a valid subset
    for i = start to n-1:
      current.append(nums[i]) // CHOOSE
      backtrack(i + 1, current) // EXPLORE
      current.pop() // UN-CHOOSE (backtrack!)
  backtrack(0, [])
  return result
Time: O(n × 2n) Space: O(n) recursion depth
Pros
  • Universal template for subset/permutation/combination problems
  • Easy to add constraints (skip duplicates, prune)
  • Elegant recursive structure
Cons
  • Requires understanding of recursion
  • Stack depth = n
Approach 3
Bit Masking

Each integer from 0 to 2n-1 represents a subset. If bit j is set in integer i, include nums[j].

function subsets(nums):
  n = len(nums)
  result = []
  for mask = 0 to 2n - 1:
    subset = []
    for j = 0 to n-1:
      if mask & (1 << j):
        subset.append(nums[j])
    result.append(subset)
  return result
Time: O(n × 2n) Space: O(n × 2n)
Pros
  • Very concise
  • No recursion
  • Maps naturally to binary representation
Cons
  • Limited to n ≤ ~20 (integer size)
  • Can't easily add constraints

💻 Optimal Implementation

class Solution {
public:
    vector<vector<int>> subsets(vector<int>& nums) {
        vector<vector<int>> result;
        vector<int> current;
        backtrack(nums, 0, current, result);
        return result;
    }
    void backtrack(vector<int>& nums, int start,
                   vector<int>& current, vector<vector<int>>& result) {
        result.push_back(current);  // every node is a valid subset
        for (int i = start; i < nums.size(); i++) {
            current.push_back(nums[i]);    // CHOOSE
            backtrack(nums, i + 1, current, result);  // EXPLORE
            current.pop_back();            // UN-CHOOSE
        }
    }
};
class Solution:
    def subsets(self, nums):
        result = []
        def backtrack(start, current):
            result.append(current[:])  # every node is a valid subset
            for i in range(start, len(nums)):
                current.append(nums[i])    # CHOOSE
                backtrack(i + 1, current)  # EXPLORE
                current.pop()              # UN-CHOOSE
        backtrack(0, [])
        return result
