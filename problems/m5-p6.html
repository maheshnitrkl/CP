N-Queens

Hard 🔗 LeetCode

📜 Problem Statement

Place n queens on an n×n chessboard so that no two queens attack each other (no two queens share same row, column, or diagonal). Return all distinct solutions.

Example (n=4): Two solutions exist.

🔢 Mathematical Foundation

Constraint: No two queens on same row, column, or diagonal

Rows: Place exactly 1 queen per row (guaranteed by our algorithm).

Columns: Track used columns in a set.

Diagonals: For the ↘ diagonal, all cells have the same (row - col) value. For the ↙ diagonal, all cells have the same (row + col) value. Track these in sets.

Complexity: O(n!) placements to try

💡 Approach Progression: Brute Force → Optimal

Approach 1
Try All Positions

Place queens in every possible position on the board and check validity after all n queens are placed.

function solveNQueens(n):
  for each of nn placements:
    if isValid(placement):
      result.append(placement)
Time: O(nn) Space: O(n2)
Pros
  • Exhaustive — finds all solutions
Cons
  • nn is astronomically large
  • Checks validity only after placing all queens
Why do we need a better approach?
We're placing queens in impossible positions. If row 0's queen attacks row 1's queen, there's no point placing queens in rows 2-7. We should check conflicts as we go.
Approach 2
Backtracking Row by Row with Conflict Sets

Place one queen per row. For each row, try each column. Use sets for columns, forward diagonals (row-col), and backward diagonals (row+col) to check conflicts in O(1). Backtrack immediately on conflict.

function solveNQueens(n):
  cols = Set(), diag1 = Set(), diag2 = Set()
  board = n×n grid of '..'
  function backtrack(row):
    if row == n: // all queens placed!
      result.append(board); return
    for col = 0 to n-1:
      if col in cols or (row-col) in diag1 or (row+col) in diag2:
        continue // PRUNE!
      board[row][col] = 'Q'
      cols.add(col); diag1.add(row-col); diag2.add(row+col)
      backtrack(row + 1)
      board[row][col] = '.' // UN-CHOOSE
      cols.remove(col); diag1.remove(row-col); diag2.remove(row+col)
  backtrack(0)
Time: O(n!) Space: O(n2)
Pros
  • Prunes impossible branches immediately
  • O(1) conflict check using sets
  • Row-by-row guarantees no row conflicts
Cons
  • Still exponential for large n
  • Requires understanding of diagonal indexing

💻 Optimal Implementation

class Solution {
public:
    vector<vector<string>> solveNQueens(int n) {
        vector<vector<string>> result;
        vector<string> board(n, string(n, '.'));
        set<int> cols, diag1, diag2;
        backtrack(0, n, board, cols, diag1, diag2, result);
        return result;
    }
    void backtrack(int row, int n, vector<string>& board,
                   set<int>& cols, set<int>& d1, set<int>& d2,
                   vector<vector<string>>& res) {
        if (row == n) { res.push_back(board); return; }
        for (int col = 0; col < n; col++) {
            if (cols.count(col) || d1.count(row-col) || d2.count(row+col))
                continue;
            board[row][col] = 'Q';
            cols.insert(col); d1.insert(row-col); d2.insert(row+col);
            backtrack(row+1, n, board, cols, d1, d2, res);
            board[row][col] = '.';
            cols.erase(col); d1.erase(row-col); d2.erase(row+col);
        }
    }
};
class Solution:
    def solveNQueens(self, n):
        result = []
        cols, diag1, diag2 = set(), set(), set()
        board = [['.' for _ in range(n)] for _ in range(n)]
        
        def backtrack(row):
            if row == n:
                result.append([''.join(r) for r in board])
                return
            for col in range(n):
                if col in cols or (row-col) in diag1 or (row+col) in diag2:
                    continue
                board[row][col] = 'Q'
                cols.add(col); diag1.add(row-col); diag2.add(row+col)
                backtrack(row + 1)
                board[row][col] = '.'
                cols.discard(col); diag1.discard(row-col); diag2.discard(row+col)
        
        backtrack(0)
        return result
