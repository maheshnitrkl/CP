Group Anagrams

Medium 🔗 LeetCode

📜 Problem Statement

Group strings that are anagrams of each other.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Nested Loops Comparison

Compare every string with every other string by sorting them. If they match, group them and mark them as visited.

function groupAnagrams(strs):
  visited = set(), res = []
  for i = 0 to len(strs)-1:
    if i in visited: continue
    group = [strs[i]]
    for j = i+1 to len(strs)-1:
      if sort(strs[i]) == sort(strs[j]):
        group.append(strs[j])
        visited.add(j)
    res.append(group)
  return res
Time: O(n2 × k log k) Space: O(n)
Pros
  • Straightforward logic
Cons
  • Extremely slow (O(n²))
  • Lots of redundant sorting
Why optimize?
We can avoid comparing every pair by using a Hash Map to group items that share the same sorted signature directly.
Approach 2
Hash Map with Sorted Key

Sort each string. Anagrams will have the same sorted representation. Use it as a Hash Map key.

function groupAnagrams(strs):
  map = HashMap()
  for s in strs:
    key = sort(s)
    map[key].append(s)
  return map.values()
Time: O(n × k log k) Space: O(n × k)
Pros
  • Elegant mapping
  • Very practical in most languages
Cons
  • Sorting takes O(k log k) per string
Why optimize?
If the string lengths are very large, the O(k log k) sorting dominates. We can create an anagram signature in O(k) time instead.
Approach 3
Hash Map with Count Array Key

Instead of sorting, count character frequencies (an array of size 26). Convert this array to a string or tuple and use it as the hash key.

function groupAnagrams(strs):
  map = HashMap()
  for s in strs:
    count = [0] * 26
    for c in s: count[c - 'a'] += 1
    key = tuple(count) // or string join
    map[key].append(s)
  return map.values()
Time: O(n × k) Space: O(n × k)
Pros
  • Optimal O(n × k) time
  • No sorting required
Cons
  • Converting array to tuple/string adds minor overhead

Init Map = {}

3. "tan" → sort → "ant". Map["ant"] = ["tan"]
4. "ate" → sort → "aet". Map["aet"] = ["eat", "tea", "ate"]
5. "nat" → sort → "ant". Map["ant"] = ["tan", "nat"]
6. "bat" → sort → "abt". Map["abt"] = ["bat"]

Result: [["eat", "tea", "ate"], ["tan", "nat"], ["bat"]]

💻 Optimal Implementation

class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        unordered_map<string, vector<string>> mp;
        for (string s : strs) {
            string key = s;
            sort(key.begin(), key.end());
            mp[key].push_back(s);
        }
        vector<vector<string>> res;
        for (auto& pair : mp) res.push_back(pair.second);
        return res;
    }
};
from collections import defaultdict
class Solution:
    def groupAnagrams(self, strs):
        mp = defaultdict(list)
        for s in strs:
            mp[tuple(sorted(s))].append(s)
        return list(mp.values())
