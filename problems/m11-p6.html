Binary Tree Right Side View

Medium 🔗 LeetCode

📜 Problem Statement

Given the root of a binary tree, imagine yourself standing on the right side of it, return the values of the nodes you can see ordered from top to bottom.

📚 Logic & Core Concept

Imagine standing on the right side of a binary tree. The nodes you can see form the right side view.

This translates to finding the rightmost node at each depth level of the tree.

Result[level] = Last Node visited at that level (if scanning Left to Right)
Result[level] = First Node visited at that level (if scanning Right to Left)

💡 Approach Progression: Intuitive → Optimal

Approach 1
Breadth-First Search (Level Order)

We traverse the tree level by level using a Queue. For each level, we iterate through all its nodes. The very last node we process in each level is the one visible from the right side.

function rightSideViewBFS(root):
  if root is null: return []
  queue = [root], result = []
  while queue is not empty:
    levelSize = len(queue)
    for i from 0 to levelSize - 1:
      node = queue.dequeue()
      if i == levelSize - 1: // Last node in current level
        result.append(node.val)
      if node.left: queue.enqueue(node.left)
      if node.right: queue.enqueue(node.right)
  return result
Time: O(n) Space: O(w) (Max width of tree)
Pros
  • Very intuitive, directly maps to "levels"
Cons
  • Takes O(N/2) space for perfectly balanced trees (the bottom level)
Why optimize?
Can we do it recursively with Depth-First Search? Yes, by being smart about the order we visit nodes.
Approach 2
Depth-First Search (Root → Right → Left)

We traverse using a modified Preorder: we visit the Root, then the Right child, then the Left child. By passing the current depth as an argument, we know if we are visiting a level for the first time. If depth == result.length, we add the node!

function rightSideViewDFS(root):
  result = []
  function dfs(node, depth):
    if node is null: return
    if depth == len(result):
      result.append(node.val) // First node we see at this depth!
    dfs(node.right, depth + 1) // Go RIGHT first
    dfs(node.left, depth + 1) // Then go LEFT
  dfs(root, 0)
  return result
Time: O(n) Space: O(h) (Height of tree for call stack)
Pros
  • Often takes less memory than BFS for bushy trees
  • Code is extremely short and elegant
Cons
  • Call stack overflow risk for skewed trees

💻 Optimal Implementation

class Solution {
    void dfs(TreeNode* node, int depth, vector<int>& result) {
        if (node == nullptr) return;
        
        // If we're visiting this depth for the first time
        if (depth == result.size()) {
            result.push_back(node->val);
        }
        
        // Go right first, so the first node we see at each depth is the rightmost
        dfs(node->right, depth + 1, result);
        dfs(node->left, depth + 1, result);
    }
    
public:
    vector<int> rightSideView(TreeNode* root) {
        vector<int> result;
        dfs(root, 0, result);
        return result;
    }
};
class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> List[int]:
        result = []
        
        def dfs(node, depth):
            if not node:
                return
            
            # If we're visiting this depth for the first time
            if depth == len(result):
                result.append(node.val)
                
            # Go right first, so the first node we see at each depth is the rightmost
            dfs(node.right, depth + 1)
            dfs(node.left, depth + 1)
            
        dfs(root, 0)
        return result
