Path Sum

Easy 🔗 LeetCode

📜 Problem Statement

Given the root of a binary tree and an integer targetSum, return true if the tree has a root-to-leaf path such that adding up all the values along the path equals targetSum.

📚 Logic & Core Concept

You are given the root of a binary tree and an integer targetSum.

The goal is to determine if the tree has a root-to-leaf path such that adding up all the values along the path equals targetSum.

A leaf is a node with NO children.

Key Insight: As we traverse down the tree, we can subtract the current node's value from the targetSum. If we reach a leaf node and the remaining sum equals 0, we found a valid path!

💡 Approach Progression: Recursion → Iterative

Approach 1
Recursion (DFS)

The most elegant approach. At each node, subtract its value from targetSum. Check if it's a leaf node. If so, return whether the remaining sum is exactly 0. If not, recursively check the left and right children with the new sum.

function hasPathSum(root, targetSum):
  if root is null: return false
  targetSum -= root.val
  if root.left is null and root.right is null: // Leaf node!
    return targetSum == 0
  return hasPathSum(root.left, targetSum) or hasPathSum(root.right, targetSum)
Time: O(n) Space: O(h)
Pros
  • Short, highly readable
Cons
  • Call stack usage for skewed trees
Why explore alternatives?
To avoid call stack overflow in deep trees, we can use iterative DFS (Stack) or iterative BFS (Queue).
Approach 2
Iterative DFS (Stack)

We use a stack to simulate recursion. The stack stores pairs: (node, current_remaining_sum). We pop a node, subtract its value, and if it's a leaf and the sum is 0, we found it.

function hasPathSumDFS(root, targetSum):
  if root is null: return false
  stack = [(root, targetSum - root.val)]
  while stack is not empty:
    node, currSum = stack.pop()
    if node is leaf and currSum == 0: return true
    if node.right: stack.push((node.right, currSum - node.right.val))
    if node.left: stack.push((node.left, currSum - node.left.val))
  return false
Time: O(n) Space: O(h)
Pros
  • Safe from stack overflow
Cons
  • Manual stack management makes it slightly more verbose
Approach 3
Breadth-First Search (Queue)

Similar to iterative DFS, but we use a Queue. This explores the tree level-by-level, carrying the remaining sum down to the leaves.

function hasPathSumBFS(root, targetSum):
  if root is null: return false
  queue = [(root, targetSum - root.val)]
  while queue is not empty:
    node, currSum = queue.dequeue()
    if node is leaf and currSum == 0: return true
    if node.left: queue.enqueue((node.left, currSum - node.left.val))
    if node.right: queue.enqueue((node.right, currSum - node.right.val))
  return false
Time: O(n) Space: O(w) (Max width of tree)

💻 Optimal Implementation

class Solution {
public:
    bool hasPathSum(TreeNode* root, int targetSum) {
        if (root == nullptr) {
            return false;
        }
        
        targetSum -= root->val;
        
        // If we reach a leaf node
        if (root->left == nullptr && root->right == nullptr) {
            return targetSum == 0;
        }
        
        // Check recursively in left and right subtrees
        return hasPathSum(root->left, targetSum) || hasPathSum(root->right, targetSum);
    }
};
class Solution:
    def hasPathSum(self, root: Optional[TreeNode], targetSum: int) -> bool:
        if not root:
            return False
            
        targetSum -= root.val
        
        # If we reach a leaf node
        if not root.left and not root.right:
            return targetSum == 0
            
        # Check recursively in left and right subtrees
        return self.hasPathSum(root.left, targetSum) or \
               self.hasPathSum(root.right, targetSum)
