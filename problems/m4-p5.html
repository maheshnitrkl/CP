Kth Largest Element in an Array

Medium 🔗 LeetCode

📜 Problem Statement

Given an integer array nums and an integer k, return the kth largest element.

Example: [3,2,1,5,6,4], k=2 → 5

🔢 Mathematical Foundation

kth largest = (n-k)th smallest (0-indexed)

Quickselect (based on Quick Sort's partition) can find the k-th element in O(n) average time without fully sorting.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Full Sort

Sort the entire array and return the element at index n-k.

function findKthLargest(nums, k):
  sort(nums)
  return nums[n - k]
Time: O(n log n) Space: O(1)
Pros
  • One line solution
  • Easy to understand
Cons
  • Sorts the entire array when we only need one element
Why do we need a better approach?
We're sorting everything when we only need the k-th element. Quickselect can find it in O(n) average by partial sorting.
Approach 2
Min-Heap of Size k

Maintain a min-heap of size k. For each element, push it. If heap size > k, pop the smallest. The top of the heap is the k-th largest.

function findKthLargest(nums, k):
  minHeap = MinHeap()
  for x in nums:
    minHeap.push(x)
    if minHeap.size() > k:
      minHeap.pop()
  return minHeap.top()
Time: O(n log k) Space: O(k)
Pros
  • O(n log k) is better than O(n log n) when k << n
  • Works for streaming data
Cons
  • Still uses O(k) space
  • Not O(n) time
Why do we need a better approach?
Quickselect achieves O(n) average time with O(1) extra space.
Approach 3
Quickselect (Hoare's Selection)

Use Quick Sort's partition function. After partitioning, if pivot is at index n-k, we're done. If pivot < n-k, recurse right. Otherwise, recurse left. Average O(n).

function findKthLargest(nums, k):
  target = n - k
  lo = 0, hi = n-1
  while true:
    pivotIdx = partition(nums, lo, hi)
    if pivotIdx == target: return nums[pivotIdx]
    elif pivotIdx < target: lo = pivotIdx + 1
    else: hi = pivotIdx - 1
Time: O(n) average, O(n2) worst Space: O(1)
Pros
  • O(n) average time
  • O(1) extra space
  • Only processes relevant partition
Cons
  • O(n^2) worst case (mitigated by random pivot)
  • Modifies the input array

💻 Optimal Implementation

class Solution {
public:
    int findKthLargest(vector<int>& nums, int k) {
        int target = nums.size() - k;
        int lo = 0, hi = nums.size() - 1;
        while (true) {
            int pivot = partition(nums, lo, hi);
            if (pivot == target) return nums[pivot];
            else if (pivot < target) lo = pivot + 1;
            else hi = pivot - 1;
        }
    }
    int partition(vector<int>& nums, int lo, int hi) {
        int pivot = nums[hi], i = lo;
        for (int j = lo; j < hi; j++) {
            if (nums[j] <= pivot) swap(nums[i++], nums[j]);
        }
        swap(nums[i], nums[hi]);
        return i;
    }
};
import random
class Solution:
    def findKthLargest(self, nums, k):
        target = len(nums) - k
        def quickselect(lo, hi):
            pivot_idx = random.randint(lo, hi)
            nums[pivot_idx], nums[hi] = nums[hi], nums[pivot_idx]
            pivot, i = nums[hi], lo
            for j in range(lo, hi):
                if nums[j] <= pivot:
                    nums[i], nums[j] = nums[j], nums[i]
                    i += 1
            nums[i], nums[hi] = nums[hi], nums[i]
            if i == target: return nums[i]
            elif i < target: return quickselect(i + 1, hi)
            else: return quickselect(lo, i - 1)
        return quickselect(0, len(nums) - 1)
