Linked List Cycle

Easy 🔗 LeetCode

📜 Problem Statement

Given a linked list, determine if it has a cycle.

🔢 Mathematical Foundation

Floyd's Cycle Detection (Tortoise & Hare)

Slow moves 1 step, fast moves 2 steps. If a cycle exists, fast will eventually catch slow inside the cycle (they meet). If no cycle, fast reaches null.

Why do they meet? Once both are in the cycle, the gap decreases by 1 each step (fast gains 1 step on slow). So they must meet within one full cycle traversal.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Hash Set

Track visited nodes. If we visit the same node twice, there's a cycle.

function hasCycle(head):
  visited = Set()
  while head:
    if head in visited: return true
    visited.add(head); head = head.next
  return false
Time: O(n) Space: O(n)
Pros
  • Simple and correct
Cons
  • O(n) extra space
Why do we need a better approach?
Floyd's algorithm detects cycles in O(1) space.
Approach 2
Floyd's Slow & Fast Pointers

Slow moves 1 step, fast moves 2 steps. If they meet, cycle exists. If fast reaches null, no cycle.

function hasCycle(head):
  slow = fast = head
  while fast AND fast.next:
    slow = slow.next
    fast = fast.next.next
    if slow == fast: return true
  return false
Time: O(n) Space: O(1)
Pros
  • O(1) space
  • Elegant
  • Classic algorithm
Cons
  • Doesn't find the cycle start (need Linked List Cycle II for that)

👀 Step-by-Step Walkthrough

Trace: 3 → 2 → 0 → -4 → (cycle to 2)
Initialize: slow = 3, fast = 3

Step 1:
  slow moves 1 step to 2
  fast moves 2 steps to 0

Step 2:
  slow moves 1 step to 0
  fast moves 2 steps to 2 (from -4 → 2)

Step 3:
  slow moves 1 step to -4
  fast moves 2 steps to -4 (from 2 → 0 → -4)

slow == fast (-4 == -4). Cycle Detected! Return true.

💻 Optimal Implementation

class Solution {
public:
    bool hasCycle(ListNode *head) {
        ListNode *slow = head, *fast = head;
        while (fast && fast->next) {
            slow = slow->next;
            fast = fast->next->next;
            if (slow == fast) return true;
        }
        return false;
    }
};
class Solution:
