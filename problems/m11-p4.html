Maximum Depth of Binary Tree

Easy 🔗 LeetCode

📜 Problem Statement

Given the root of a binary tree, return its maximum depth. A binary tree's maximum depth is the number of nodes along the longest path from the root node down to the farthest leaf node.

📚 Logic & Core Concept

The maximum depth of a binary tree is the number of nodes along the longest path from the root node down to the farthest leaf node.

This problem can be broken down recursively:

Depth(node) = 1 + max( Depth(node.left), Depth(node.right) )

Base case: If the node is null, its depth is 0.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Recursion (DFS)

The most intuitive approach. We ask the left child for its max depth, ask the right child for its max depth, take the maximum of the two, and add 1 for the current node.

function maxDepth(root):
  if root is null: return 0
  leftDepth = maxDepth(root.left)
  rightDepth = maxDepth(root.right)
  return 1 + max(leftDepth, rightDepth)
Time: O(n) Space: O(h)
Pros
  • Shortest and cleanest code
Cons
  • Call stack usage can be high for skewed trees
Why optimize?
Recursion is generally optimal here, but in production systems with limited call stack sizes, an iterative approach might be preferred.
Approach 2
Iterative DFS (Stack)

We can simulate the call stack using our own stack data structure. Each item in the stack will store a pair: (node, current_depth). We update a global max depth as we pop items.

function maxDepthIterativeDFS(root):
  if root is null: return 0
  stack = [(root, 1)]
  max_d = 0
  while stack is not empty:
    node, depth = stack.pop()
    max_d = max(max_d, depth)
    if node.left: stack.push((node.left, depth + 1))
    if node.right: stack.push((node.right, depth + 1))
  return max_d
Time: O(n) Space: O(h)
Pros
  • Prevents call stack overflow
Cons
  • More verbose than recursion
Why optimize?
What if the tree is extremely wide rather than deep? DFS might use a lot of memory. We can also use Level Order Traversal (BFS).
Approach 3
Iterative BFS (Level Order / Queue)

We process the tree level by level using a Queue. Each time we move to a new level, we increment our depth counter.

function maxDepthBFS(root):
  if root is null: return 0
  queue = [root], depth = 0
  while queue is not empty:
    levelSize = len(queue)
    for i from 0 to levelSize - 1:
      node = queue.dequeue()
      if node.left: queue.enqueue(node.left)
      if node.right: queue.enqueue(node.right)
    depth += 1
  return depth
Time: O(n) Space: O(w) (Max width of tree)
Pros
  • Great if the tree is heavily imbalanced to the left or right
Cons
  • Uses more memory for perfectly balanced trees (width > height)

💻 Optimal Implementation

class Solution {
public:
    int maxDepth(TreeNode* root) {
        // Base case: empty tree has depth 0
        if (root == nullptr) {
            return 0;
        }
        
        // Recursively find depth of left and right subtrees
        int leftDepth = maxDepth(root->left);
        int rightDepth = maxDepth(root->right);
        
        // Depth is max of left and right, plus 1 for the current node
        return 1 + max(leftDepth, rightDepth);
    }
};
class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        # Base case: empty tree has depth 0
        if not root:
            return 0
            
        # Recursively find depth of left and right subtrees
        left_depth = self.maxDepth(root.left)
        right_depth = self.maxDepth(root.right)
        
        # Depth is max of left and right, plus 1 for the current node
        return 1 + max(left_depth, right_depth)
