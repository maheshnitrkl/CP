Non Repeating Numbers

Medium 🔗 GFG

📜 Problem Statement

Given an array where every element appears twice except for two elements, find those two non-repeating elements. Return them in sorted order.

Example: arr = [1, 2, 3, 2, 1, 4] → Output: [3, 4]

🔢 Mathematical Foundation

XOR(all) = a ^ b (the two unique numbers)

Since all paired elements cancel out, XOR of the entire array = a ^ b.

Key Insight: Find any set bit in (a ^ b). This bit differs between a and b.

Use this differing bit to partition the array into two groups. XOR each group separately to isolate a and b.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Hash Map Frequency Count

Count the frequency of each element using a hash map. Return elements with frequency 1.

function findTwoUnique(arr):
  freq = HashMap()
  for x in arr:
    freq[x] += 1
  result = []
  for key, count in freq:
    if count == 1:
      result.append(key)
  return sort(result)
Time: O(n) Space: O(n)
Pros
  • Easy to understand
  • Works for any number of unique elements
Cons
  • O(n) extra space
  • Not leveraging the structure of the problem (exactly pairs)
Why do we need a better approach?
We're using O(n) space. The constraint that elements appear exactly twice means we can use XOR to do this in O(1) space.
Approach 2
XOR with Bit Partitioning

XOR the entire array to get a ^ b. Find the rightmost set bit (this bit is different between a and b). Partition all elements by this bit and XOR each group. Each group contains exactly one of the unique numbers.

function findTwoUnique(arr):
  xorAll = 0
  for x in arr: xorAll ^= x
  
  diffBit = xorAll & (-xorAll) // rightmost set bit
  
  a = 0, b = 0
  for x in arr:
    if x & diffBit:
      a ^= x // group with this bit set
    else:
      b ^= x // group without this bit
  return sort([a, b])
Time: O(n) Space: O(1)
Pros
  • O(1) space
  • Two passes through the array
  • Elegant XOR technique
Cons
  • Harder to understand
  • Requires understanding of 2's complement for rightmost set bit

👀 Step-by-Step Walkthrough

Trace: arr = [1, 2, 3, 2, 1, 4]
Pass 1: XOR all = 1^2^3^2^1^4 = 3^4 = 7 (111)
Rightmost set bit: 7 & (-7) = 7 & (...11111001) = 1 (001)

Pass 2 (partition by bit 0):
  Bit 0 SET: {1, 3, 1} → XOR = 1^3^1 = 3
  Bit 0 NOT SET: {2, 2, 4} → XOR = 2^2^4 = 4
Result: [3, 4]

💻 Optimal Implementation

class Solution {
public:
    vector<int> singleNumber(vector<int>& nums) {
        int xorAll = 0;
        for (int x : nums) xorAll ^= x;
        
        int diffBit = xorAll & (-xorAll); // rightmost set bit
        int a = 0, b = 0;
        for (int x : nums) {
            if (x & diffBit) a ^= x;
            else b ^= x;
        }
        return a < b ? vector<int>{a, b} : vector<int>{b, a};
    }
};
