Jump Game III

Medium 🔗 Problem Link

📜 Problem Statement

Given an array of non-negative integers arr, you are initially positioned at start index of the array. When you are at index i, you can jump to i + arr[i] or i - arr[i], check if you can reach to any index with value 0.

📚 Logic & Core Concept

You are given an array of non-negative integers arr and a start index. When you are at index i, you can jump to i + arr[i] or i - arr[i].

The goal is to determine if you can reach any index with value 0.

Graph Interpretation:
  • Nodes: The indices of the array (0 to n-1).
  • Edges: Directed edges from i to i + arr[i] and i - arr[i] (if within bounds).
  • Destination: Any node j such that arr[j] == 0.

💡 Approach Progression: DFS → BFS

Approach 1
Depth-First Search (DFS)

We can solve this with recursion. From index i, we recursively try to solve for i + arr[i] and i - arr[i]. To avoid infinite loops (cycles), we can use a visited set, or trickily, mutate the array by making arr[i] negative after visiting it.

function canReach(arr, start):
  if start < 0 or start >= len(arr) or arr[start] < 0:
    return false
  
  if arr[start] == 0:
    return true
  
  // Mark as visited by negating
  arr[start] = -arr[start]
  
  return canReach(arr, start + arr[start]) or canReach(arr, start - arr[start])
Time: O(N) Space: O(N) (Recursion Stack)
Pros
  • Extremely concise code.
  • Mutating the array saves O(N) space for the visited set (though the call stack is still O(N)).
Cons
  • Mutates input (not always acceptable in production).
  • Deep recursion can cause stack overflow in some languages.
Approach 2
Breadth-First Search (BFS)

We can iteratively explore level by level using a Queue. We start by putting the start index in the queue. While the queue isn't empty, pop an index, check if it's 0. If not, calculate left and right jumps and add valid ones to the queue.

function canReachBFS(arr, start):
  queue = [start]
  
  while queue is not empty:
    curr = queue.dequeue()
    
    if arr[curr] == 0: return true
    if arr[curr] < 0: continue
    
    // Left Jump
    if curr - arr[curr] >= 0: queue.enqueue(curr - arr[curr])
    // Right Jump
    if curr + arr[curr] < len(arr): queue.enqueue(curr + arr[curr])
    
    // Mark visited
    arr[curr] = -arr[curr]
  
  return false
Time: O(N) Space: O(N) (Queue)

💻 Optimal Implementation

class Solution {
public:
    bool canReach(vector<int>& arr, int start) {
        // Out of bounds or already visited
        if (start < 0 || start >= arr.size() || arr[start] < 0) {
            return false;
        }
        
        // Target found
        if (arr[start] == 0) {
            return true;
        }
        
        // Mark as visited by making it negative
        arr[start] = -arr[start];
        
        // Jump left or right
        return canReach(arr, start + arr[start]) || canReach(arr, start - arr[start]);
    }
};
class Solution:
    def canReach(self, arr: List[int], start: int) -> bool:
        if start < 0 or start >= len(arr) or arr[start] < 0:
            return False
            
        if arr[start] == 0:
            return True
            
        # Mark as visited
        arr[start] = -arr[start]
        
        # Jump right or left
        return self.canReach(arr, start + arr[start]) or self.canReach(arr, start - arr[start])
