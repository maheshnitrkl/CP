Longest Increasing Subsequence

Medium 🔗 LeetCode

📜 Problem Statement

Given an integer array nums, return the length of the longest strictly increasing subsequence.

📚 Mathematical Foundation

Let dp[i] be the length of the longest strictly increasing subsequence that ends exactly at index i.

For any element nums[i], we can extend an existing increasing subsequence that ends at some index j < i, provided that nums[j] < nums[i].

dp[i] = 1 + max(dp[j]) for all 0 ≤ j < i where nums[j] < nums[i]

Base case: Every element itself is an increasing subsequence of length 1, so dp[i] = 1 initially.

The overall answer is the maximum value in the entire dp array.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Brute Force Recursion

At each element, we can either include it (if it's greater than the previous included element) or exclude it. Find the max of both branches.

function LIS(i, prevIdx, nums):
  if i == len(nums): return 0
  take = 0
  if prevIdx < 0 or nums[i] > nums[prevIdx]:
    take = 1 + LIS(i+1, i, nums)
  skip = LIS(i+1, prevIdx, nums)
  return max(take, skip)
Time: O(2n) Space: O(n)
Pros
  • Explores all possibilities safely
Cons
  • Exponential time, TLE
Why optimize?
Many subproblems (index, prevIndex) overlap. We can cache these!
Approach 2
Top-Down Memoization

Use a 2D array memo[index][prevIndex+1] to cache the recursion results.

function LIS(i, prevIdx, memo):
  if i == n: return 0
  if memo[i][prevIdx+1] exists: return memo[i][prevIdx+1]
  ... // same logic as recursion
  memo[i][prevIdx+1] = max(take, skip)
  return memo[i][prevIdx+1]
Time: O(n2) Space: O(n2)
Pros
  • Fast pseudo-polynomial time
Cons
  • 2D array wastes memory for this problem
Why optimize?
We can do this bottom-up with a 1D DP array based on the mathematical formulation, reducing space to O(n).
Approach 3
1D DP Tabulation

Build an array dp where dp[i] stores the LIS ending at index i. Compare nums[i] with all previous elements to update dp[i].

function lengthOfLIS(nums):
  dp = array of size N filled with 1s
  for i = 1 to N-1:
    for j = 0 to i-1:
      if nums[i] > nums[j]:
        dp[i] = max(dp[i], dp[j] + 1)
  return max(dp)
Time: O(n2) Space: O(n)
Pros
  • Clean, minimal space
Cons
  • O(n2) time is slow for large arrays
Why optimize?
Instead of a linear scan to find a subsequence to extend, we can maintain an active list of subsequence tails and use Binary Search!
Approach 4
Patience Sorting with Binary Search

Maintain an array tails where tails[i] stores the smallest tail of all increasing subsequences of length i+1. Since tails is always sorted, we can use binary search.

function lengthOfLIS(nums):
  tails = []
  for x in nums:
    idx = binarySearch(tails, x) // first element ≥ x
    if idx == len(tails): tails.push(x)
    else: tails[idx] = x
  return len(tails)
Time: O(n log n) Space: O(n)
Pros
  • Theoretically fastest possible
Cons
  • Less intuitive than DP

💻 Optimal Implementation

class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        vector<int> tails;
        
        for (int x : nums) {
            // lower_bound does binary search to find the first element ≥ x
            auto it = lower_bound(tails.begin(), tails.end(), x);
            
            if (it == tails.end()) {
                // x is larger than all elements, append to extend LIS
                tails.push_back(x);
            } else {
                // overwrite the element to maintain smaller tails
                *it = x;
            }
        }
        
        return tails.size();
    }
};
import bisect

class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        tails = []
        
        for x in nums:
            # bisect_left finds insertion point to maintain sorted order
            idx = bisect.bisect_left(tails, x)
            
            if idx == len(tails):
                # x is larger than all elements, append to extend LIS
                tails.append(x)
            else:
                # overwrite the element to maintain smaller tails
                tails[idx] = x
                
        return len(tails)
