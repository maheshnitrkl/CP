Permutations II

Medium 🔗 LeetCode

📜 Problem Statement

Given a collection of numbers that might contain duplicates, return all unique permutations.

Example: [1,1,2] → [[1,1,2],[1,2,1],[2,1,1]]

🔢 Mathematical Foundation

Unique Permutations = n! / (k1! × k2! × ... × km!)

Where ki is the count of each distinct element. For [1,1,2]: 3! / (2! × 1!) = 3.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Generate All + Set Filter

Generate all n! permutations, use a set to filter duplicates.

function permuteUnique(nums):
  allPerms = generateAll(nums)
  return removeDuplicates(allPerms)
Time: O(n × n!) Space: O(n × n!)
Pros
  • Simple logic
Cons
  • Generates many duplicates just to throw them away
  • Wastes time and memory
Why do we need a better approach?
We're generating duplicates and filtering. Better to avoid generating duplicates in the first place.
Approach 2
Sort + Skip Duplicates in Backtracking

Sort the array first. During backtracking, skip nums[i] if it equals nums[i-1] and nums[i-1] wasn't used at the same recursion level.

function permuteUnique(nums):
  sort(nums) // KEY: sort first!
  result = [], used = [false] * n
  function backtrack(current):
    if len(current) == n:
      result.append(copy(current)); return
    for i = 0 to n-1:
      if used[i]: continue
      if i > 0 AND nums[i] == nums[i-1] AND !used[i-1]:
        continue // skip duplicate!
      used[i] = true; current.append(nums[i])
      backtrack(current)
      current.pop(); used[i] = false
  backtrack([]); return result
Time: O(n × n!) Space: O(n)
Pros
  • No duplicate generation
  • Pruning eliminates entire subtrees
  • Same template as Permutations I with one extra line
Cons
  • Must sort first
  • The duplicate-skip condition can be tricky to understand

👀 Step-by-Step Walkthrough

Why does the skip condition work? nums = [1, 1, 2] (sorted)
When building permutations at the same recursion level:

i=0: use nums[0]=1 → OK, explore
i=1: nums[1]=1 == nums[0]=1, and used[0]=false → SKIP!
    (If we don't skip, we'd generate the same subtrees as i=0)
i=2: use nums[2]=2 → OK, explore

The rule: "If the previous identical element wasn't used at this level, skip" ensures that among identical elements, we always pick the leftmost unused one first. This guarantees each unique permutation is generated exactly once.

💻 Optimal Implementation

class Solution {
public:
    vector<vector<int>> permuteUnique(vector<int>& nums) {
        sort(nums.begin(), nums.end());
        vector<vector<int>> result;
        vector<int> current;
        vector<bool> used(nums.size(), false);
        backtrack(nums, used, current, result);
        return result;
    }
    void backtrack(vector<int>& nums, vector<bool>& used,
                   vector<int>& cur, vector<vector<int>>& res) {
        if (cur.size() == nums.size()) { res.push_back(cur); return; }
        for (int i = 0; i < nums.size(); i++) {
            if (used[i]) continue;
            if (i > 0 && nums[i] == nums[i-1] && !used[i-1]) continue;
            used[i] = true; cur.push_back(nums[i]);
            backtrack(nums, used, cur, res);
            cur.pop_back(); used[i] = false;
        }
    }
};
class Solution:
    def permuteUnique(self, nums):
        nums.sort()
        result, used = [], [False] * len(nums)
        def backtrack(current):
            if len(current) == len(nums):
                result.append(current[:]); return
            for i in range(len(nums)):
                if used[i]: continue
                if i > 0 and nums[i] == nums[i-1] and not used[i-1]: continue
                used[i] = True; current.append(nums[i])
                backtrack(current)
                current.pop(); used[i] = False
        backtrack([])
        return result
