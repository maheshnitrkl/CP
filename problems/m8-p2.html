Min Stack

Medium 🔗 LeetCode

📜 Problem Statement

Design a stack that supports push, pop, top, and retrieving the minimum element in constant time.

🔢 Mathematical Foundation

State Tracking

To get the minimum in O(1) time, every element in the stack must remember what the minimum was at the time it was pushed.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Search on getMin()

Push/Pop are O(1). For getMin(), search the entire stack.

function getMin():
  minVal = infinity
  for x in stack:
    minVal = min(minVal, x)
  return minVal
Time: O(1) push, O(n) min Space: O(n)
Pros
  • Simple
Cons
  • O(n) for getMin breaks the O(1) requirement
Why do we need a better approach?
We must store the minimum at each step to retrieve it in O(1).
Approach 2
Pair Stack

Store pairs (value, current_min) in the stack. When pushing, current_min = min(val, previous_min).

class MinStack:
  stack = []
  
  function push(val):
    minVal = stack.empty() ? val : min(val, stack.top().min)
    stack.push({val, minVal})
  
  function pop(): stack.pop()
  function top(): return stack.top().val
  function getMin(): return stack.top().min
Time: O(1) all ops Space: O(n)
Pros
  • All operations O(1)
  • Elegant state tracking
Cons
  • Slightly more memory per element (storing pairs)

💻 Optimal Implementation

class MinStack {
    stack<pair<int, int>> st;
public:
    void push(int val) {
        if (st.empty()) st.push({val, val});
        else st.push({val, min(val, st.top().second)});
    }
    void pop() { st.pop(); }
    int top() { return st.top().first; }
    int getMin() { return st.top().second; }
};
class MinStack:
    def __init__(self):
        self.st = []
    def push(self, val):
        cur_min = self.st[-1][1] if self.st else val
        self.st.append((val, min(val, cur_min)))
    def pop(self):
        self.st.pop()
    def top(self):
        return self.st[-1][0]
    def getMin(self):
        return self.st[-1][1]
