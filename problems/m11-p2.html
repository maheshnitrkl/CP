Binary Tree Inorder Traversal

Easy 🔗 LeetCode

📜 Problem Statement

Given the root of a binary tree, return the inorder traversal of its nodes' values.

📚 Logic & Core Concept

Inorder Traversal visits the nodes of a binary tree in the following order:

  1. Traverse the Left Subtree.
  2. Visit the Current Node (Root).
  3. Traverse the Right Subtree.
Order: Left → Node → Right (LNR)

Key Insight: In a Binary Search Tree (BST), inorder traversal visits nodes in strictly increasing sorted order!

💡 Approach Progression: Brute Force → Optimal

Approach 1
Recursive Traversal

The simplest approach relies on the system call stack to naturally defer processing the root until the left subtree is fully processed.

function inorder(node, result):
  if node is null: return
  inorder(node.left, result) // Left
  result.append(node.val) // Node
  inorder(node.right, result) // Right
Time: O(n) Space: O(h) (Call Stack)
Pros
  • Trivial to implement
Cons
  • Call stack limits depth (recursion limit)
Why optimize?
Can we manually simulate the recursion so we don't rely on the system's memory-heavy call stack?
Approach 2
Iterative (Using a Stack)

We use our own Stack. We traverse as far left as possible, pushing nodes to the stack. Once we hit null, we pop from the stack, visit the node, and then move to its right child.

function inorderIterative(root):
  stack = []
  result = []
  curr = root
  while curr != null or stack is not empty:
    while curr != null:
      stack.push(curr)
      curr = curr.left
    curr = stack.pop()
    result.append(curr.val)
    curr = curr.right
  return result
Time: O(n) Space: O(h)
Pros
  • Avoids recursion depth limits
Cons
  • Still requires O(h) space for the stack
Why optimize?
We can optimize the space to O(1) by restructuring the tree temporarily to keep track of where to return.
Approach 3
Morris Traversal

We link the rightmost node of the left subtree back to the current node. This "thread" allows us to return to the parent without a stack. When we revisit the node via the thread, we remove the thread to restore the tree.

function morrisInorder(root):
  curr = root, result = []
  while curr != null:
    if curr.left == null:
      result.append(curr.val)
      curr = curr.right
    else:
      prev = curr.left
      while prev.right != null and prev.right != curr:
        prev = prev.right
      if prev.right == null: // Create thread
        prev.right = curr
        curr = curr.left
      else: // Remove thread
        prev.right = null
        result.append(curr.val)
        curr = curr.right
Time: O(n) Space: O(1)
Pros
  • Optimal O(1) extra space!
Cons
  • Mutates the tree during traversal
  • Harder to implement correctly

💻 Optimal Implementation

class Solution {
public:
    vector<int> inorderTraversal(TreeNode* root) {
        vector<int> result;
        stack<TreeNode*> s;
        TreeNode* curr = root;
        
        while (curr != nullptr || !s.empty()) {
            // Reach the left most node of the current node
            while (curr != nullptr) {
                s.push(curr);
                curr = curr->left;
            }
            
            // Current must be NULL at this point
            curr = s.top();
            s.pop();
            
            // Add the node to result
            result.push_back(curr->val);
            
            // We have visited the node and its left subtree. Now go right.
            curr = curr->right;
        }
        
        return result;
    }
};
class Solution:
    def inorderTraversal(self, root: Optional[TreeNode]) -> List[int]:
        result = []
        stack = []
        curr = root
        
        while curr or stack:
            # Reach the left most node of the current node
            while curr:
                stack.append(curr)
                curr = curr.left
                
            # Current must be None at this point
            curr = stack.pop()
            
            # Add the node to result
            result.append(curr.val)
            
            # We have visited the node and its left subtree. Now go right.
            curr = curr.right
            
        return result
