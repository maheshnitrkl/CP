Find All Duplicates in an Array

Medium 🔗 LeetCode

📜 Problem Statement

Array elements are in range [1, n]. Find all duplicates in O(n) time and O(1) space.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Nested Loops

Check every element against every other element. If they match, add to results (ensure no duplicates in results).

Time: O(n2) Space: O(1)
Pros
  • Simple
Cons
  • Time Limit Exceeded
Approach 2
Sorting

Sort the array. Duplicates will be adjacent, so we can find them in a single pass.

Time: O(n log n) Space: O(1)
Pros
  • No extra space needed
Cons
  • O(n log n) time
Approach 3
Hash Set / Map

Track seen elements in a Hash Set. If an element is already in the set, it's a duplicate.

Time: O(n) Space: O(n)
Pros
  • Fast O(n) time
Cons
  • Fails the problem's O(1) space constraint
Why optimize?
The problem strictly asks for O(1) space. Since elements are in the range [1, n], we can use the array itself as a hash map!
Approach 4
In-place Array Marking

Use the array indices as a hash map. For element x, negate the value at index abs(x)-1. If it's already negative, x is a duplicate.

function findDuplicates(nums):
  res = []
  for x in nums:
    idx = abs(x) - 1
    if nums[idx] < 0: res.append(abs(x))
    else: nums[idx] *= -1
  return res
Time: O(n) Space: O(1)
Pros
  • No extra space
  • Clever indexing trick
Cons
  • Modifies input array

💻 Optimal Implementation

class Solution {
public:
    vector<int> findDuplicates(vector<int>& nums) {
        vector<int> res;
        for (int x : nums) {
            int idx = abs(x) - 1;
            if (nums[idx] < 0) res.push_back(abs(x));
            else nums[idx] *= -1;
        }
        return res;
    }
};
