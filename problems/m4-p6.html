Top K Frequent Elements

Medium 🔗 LeetCode

📜 Problem Statement

Given an integer array nums and integer k, return the k most frequent elements.

Example: [1,1,1,2,2,3], k=2 → [1,2]

🔢 Mathematical Foundation

Bucket Sort by Frequency

Maximum possible frequency = n. Create n+1 buckets where bucket[i] holds elements with frequency i. Iterate buckets from high to low to get top-k.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Sort by Frequency

Count frequencies, then sort elements by frequency in descending order. Take the first k.

function topKFrequent(nums, k):
  freq = countFrequencies(nums)
  sorted = sortByFrequency(freq, descending)
  return sorted[:k]
Time: O(n log n) Space: O(n)
Pros
  • Straightforward
  • Easy to implement
Cons
  • O(n log n) due to sorting
Why do we need a better approach?
We don't need to fully sort. A heap of size k or bucket sort can be faster.
Approach 2
Min-Heap of Size k

Count frequencies. Maintain a min-heap of size k based on frequency. The heap contains the top-k frequent elements.

function topKFrequent(nums, k):
  freq = countFrequencies(nums)
  minHeap = MinHeap() // by frequency
  for (element, count) in freq:
    minHeap.push(element, count)
    if minHeap.size() > k: minHeap.pop()
  return minHeap.elements()
Time: O(n log k) Space: O(n)
Pros
  • O(n log k) is better when k << n
Cons
  • Still requires heap operations
Why do we need a better approach?
Bucket sort can achieve O(n) by using frequency as index.
Approach 3
Bucket Sort by Frequency

Use an array of buckets where index = frequency. Iterate from highest frequency down, collecting elements until we have k.

function topKFrequent(nums, k):
  freq = countFrequencies(nums)
  buckets = array of (n+1) empty lists
  for (element, count) in freq:
    buckets[count].append(element)
  result = []
  for i = n downto 0:
    result.extend(buckets[i])
    if len(result) >= k: break
  return result[:k]
Time: O(n) Space: O(n)
Pros
  • O(n) time
  • No sorting or heap needed
  • Leverages bounded frequency range
Cons
  • Uses O(n) space for buckets

💻 Optimal Implementation

class Solution {
public:
    vector<int> topKFrequent(vector<int>& nums, int k) {
        unordered_map<int, int> freq;
        for (int x : nums) freq[x]++;
        vector<vector<int>> buckets(nums.size() + 1);
        for (auto& [num, count] : freq) buckets[count].push_back(num);
        vector<int> result;
        for (int i = buckets.size() - 1; i >= 0 && result.size() < k; i--)
            for (int num : buckets[i]) {
                result.push_back(num);
                if (result.size() == k) break;
            }
        return result;
    }
};
from collections import Counter
class Solution:
    def topKFrequent(self, nums, k):
        freq = Counter(nums)
        buckets = [[] for _ in range(len(nums) + 1)]
        for num, count in freq.items():
            buckets[count].append(num)
        result = []
        for i in range(len(buckets) - 1, -1, -1):
            result.extend(buckets[i])
            if len(result) >= k:
                return result[:k]
        return result
