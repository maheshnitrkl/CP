Intersection of Two Linked Lists

Easy 🔗 LeetCode

📜 Problem Statement

Given the heads of two singly linked lists, return the node at which they intersect. If no intersection, return null.

🔢 Mathematical Foundation

len(A) + len(B) = len(B) + len(A)

If pointer A traverses list A then list B, and pointer B traverses list B then list A, they will meet at the intersection node (or both reach null if no intersection). Both travel the same total distance!

💡 Approach Progression: Brute Force → Optimal

Approach 1
Hash Set

Store all nodes of list A in a set. Traverse list B, checking if any node is in the set.

function getIntersection(headA, headB):
  visited = Set()
  while headA: visited.add(headA); headA = headA.next
  while headB:
    if headB in visited: return headB
    headB = headB.next
  return null
Time: O(n+m) Space: O(n)
Pros
  • Simple
Cons
  • O(n) space for the set
Why do we need a better approach?
We're using O(n) space. The two-pointer technique achieves O(1) space.
Approach 2
Two Pointers (Switch Lists)

Pointer A starts at headA, pointer B at headB. When A reaches the end, redirect to headB. When B reaches the end, redirect to headA. They meet at the intersection!

function getIntersection(headA, headB):
  a = headA, b = headB
  while a != b:
    a = a.next if a else headB
    b = b.next if b else headA
  return a
Time: O(n+m) Space: O(1)
Pros
  • O(1) space
  • Elegant equal-distance property
  • Handles different-length lists automatically
Cons
  • Less intuitive why it works

💻 Optimal Implementation

class Solution {
public:
    ListNode *getIntersectionNode(ListNode *headA, ListNode *headB) {
        ListNode *a = headA, *b = headB;
        while (a != b) {
            a = a ? a->next : headB;
            b = b ? b->next : headA;
        }
        return a;
    }
};
