Contains Duplicate

Easy 🔗 LeetCode

📜 Problem Statement

Return true if any value appears at least twice.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Nested Loops

Compare every element with every other element in the array to find a match.

function containsDuplicate(nums):
  for i = 0 to len(nums)-1:
    for j = i+1 to len(nums)-1:
      if nums[i] == nums[j]: return true
  return false
Time: O(n2) Space: O(1)
Pros
  • No extra space
Cons
  • Time Limit Exceeded (TLE)
Why optimize?
Checking every pair is too slow. If we sort the array first, duplicates will be adjacent to each other.
Approach 2
Sorting

Sort the array. Then, iterate through it once to check if any adjacent elements are identical.

function containsDuplicate(nums):
  nums.sort()
  for i = 0 to len(nums)-2:
    if nums[i] == nums[i+1]: return true
  return false
Time: O(n log n) Space: O(1) or O(log n)
Pros
  • Passes all test cases
  • Minimal extra space
Cons
  • Mutates the input array
  • Slower than linear time
Why optimize?
Sorting takes O(n log n) time. We can trade space for time to achieve O(n) using a Hash Set.
Approach 3
Hash Set

Add elements to a Hash Set. If the size of the set is less than the length of the array, there are duplicates.

function containsDuplicate(nums):
  return len(Set(nums)) != len(nums)
Time: O(n) Space: O(n)
Pros
  • One liner
  • O(n) time
Cons
  • O(n) space

👀 Step-by-Step Walkthrough

Trace: nums=[1, 2, 3, 1]
Init: Set = {}

i=0: nums[0]=1. Set doesn't have 1. Add 1 → Set = {1}
i=1: nums[1]=2. Set doesn't have 2. Add 2 → Set = {1, 2}
i=2: nums[2]=3. Set doesn't have 3. Add 3 → Set = {1, 2, 3}
i=3: nums[3]=1. Set HAS 1! Collision!

Result: true (Contains duplicate)

💻 Optimal Implementation

class Solution {
public:
    bool containsDuplicate(vector<int>& nums) {
        unordered_set<int> s(nums.begin(), nums.end());
        return s.size() != nums.size();
    }
};
class Solution:
    def containsDuplicate(self, nums):
        return len(set(nums)) != len(nums)
