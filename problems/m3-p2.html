Search in Rotated Sorted Array

Medium 🔗 LeetCode

📜 Problem Statement

An array sorted in ascending order was rotated at an unknown pivot. Given the rotated array nums and a target, return its index or -1.

Example: [4,5,6,7,0,1,2], target=0 → 4

🔢 Mathematical Foundation

In a rotated sorted array, at least one half is always sorted.

If nums[lo] ≤ nums[mid], the left half [lo..mid] is sorted. Otherwise the right half [mid..hi] is sorted. Check if target lies in the sorted half; if yes, search there. Otherwise, search the other half.

💡 Approach Progression: Brute Force → Optimal

Approach 1
Linear Search

Simply scan the array to find the target.

for i = 0 to n-1:
  if nums[i] == target: return i
return -1
Time: O(n) Space: O(1)
Pros
  • Trivial to implement
Cons
  • Ignores sorted/rotated structure
  • O(n) is too slow
Why do we need a better approach?
The array is sorted (just rotated). Modified binary search can still achieve O(log n).
Approach 2
Modified Binary Search

At each step, determine which half is sorted. If the target is in the sorted range, search there. Otherwise, search the other half.

function search(nums, target):
  lo = 0, hi = n-1
  while lo ≤ hi:
    mid = lo + (hi - lo) / 2
    if nums[mid] == target: return mid
    if nums[lo] ≤ nums[mid]: // left half sorted
      if nums[lo] ≤ target < nums[mid]:
        hi = mid - 1
      else: lo = mid + 1
    else: // right half sorted
      if nums[mid] < target ≤ nums[hi]:
        lo = mid + 1
      else: hi = mid - 1
  return -1
Time: O(log n) Space: O(1)
Pros
  • O(log n) time
  • Handles rotation elegantly
  • Single pass binary search
Cons
  • Need to carefully handle boundary conditions

💻 Optimal Implementation

class Solution {
public:
    int search(vector<int>& nums, int target) {
        int lo = 0, hi = nums.size() - 1;
        while (lo <= hi) {
            int mid = lo + (hi - lo) / 2;
            if (nums[mid] == target) return mid;
            if (nums[lo] <= nums[mid]) {
                if (nums[lo] <= target && target < nums[mid]) hi = mid - 1;
                else lo = mid + 1;
            } else {
                if (nums[mid] < target && target <= nums[hi]) lo = mid + 1;
                else hi = mid - 1;
            }
        }
        return -1;
    }
};
class Solution:
    def search(self, nums, target):
        lo, hi = 0, len(nums) - 1
        while lo <= hi:
            mid = (lo + hi) // 2
            if nums[mid] == target: return mid
            if nums[lo] <= nums[mid]:
                if nums[lo] <= target < nums[mid]: hi = mid - 1
                else: lo = mid + 1
            else:
                if nums[mid] < target <= nums[hi]: lo = mid + 1
                else: hi = mid - 1
        return -1
